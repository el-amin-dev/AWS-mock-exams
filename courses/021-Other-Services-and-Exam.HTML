AWS Local Zones

Put AWS close to a city for low latency.

Quick Understanding

ConceptMeaningExam relevance
Region extensionCompute/storage near a metro.Ultra-low latency.
Opt-inYou enable the Local Zone.Not on by default.
Tied to a parent RegionManaged from that Region.Control plane.
One zoneEffectively single-AZ.Lower resilience.

Key Terms

TermSimple meaning
Local ZoneA small AWS location near a big city.
Parent RegionThe Region the Local Zone connects to.
LatencyThe delay between user and service.

Explanation

graph LR
  USER[Users in a city] -->|very low latency| LZ[Local Zone]
  LZ -->|connected to| REGION[Parent Region]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class LZ a;
  • Local Zones place compute, storage and select services close to a large city/industry center — for ultra-low latency to users there.
  • Each Local Zone connects to a parent Region; you opt-in to enable it, then extend a VPC subnet into it.
  • A Local Zone is effectively one zone (less resilient than multi-AZ in a Region). [added for exam]
  • Compare: Outposts = AWS hardware in your data center; Wavelength = edge in 5G/telco networks; CloudFront edge = caching only.

Demo

Demo · CLI v2 (read-only, free)Situation: list the Local Zones available for a Region. Check first: CLI configured.
List Local Zones for us-east-1
aws ec2 describe-availability-zones --all-availability-zones \
  --filters Name=zone-type,Values=local-zone \
  --query "AvailabilityZones[].{Zone:ZoneName,Group:GroupName,OptIn:OptInStatus}" \
  --region us-east-1 --output table
Expect: a table of Local Zones (e.g. us-east-1-bos-1 Boston) with OptInStatus. Listing is free.
Fix / Key infoTo use one you'd opt-in the group (aws ec2 modify-availability-zone-group --group-name us-east-1-bos-1 --opt-in-status opted-in) — opt-in is free, but launching resources costs money. Listing creates nothing to clean up.

Exam Pointers

Exam Tip
  • "Ultra-low latency for users in one city, but stay on AWS infra" → Local Zones.
  • Outposts = AWS racks in your DC; Wavelength = 5G edge; Local Zone = metro extension of a Region.
  • A Local Zone is single-zone — for HA you still use the parent Region's AZs.

  • Phase 1: go through all questions; answer the easy ones immediately; flag harder ones and move on. Builds confidence + banks points.
  • Phase 2: return to flagged/medium questions and give them more time.
  • Phase 3: spend remaining time on the hardest; eliminate wrong options and make your best guess — never leave a blank.
  • Time: ~65 questions in ~130 minutes ≈ 2 min each. Don't get stuck on one. [added for exam]

Question Technique

How to break down a single question.

Quick Understanding

StepWhat you doWhy
Find the askWhat is actually being asked.Often the last line.
Cut fluffRemove irrelevant wording.Less to juggle.
Spot the qualifier"cheapest", "least ops", "fastest".Picks between good answers.
EliminateDrop impossible answers.Better odds.

Key Terms

TermSimple meaning
QualifierThe word that decides "best" (cost/effort/speed).
Self-eliminatingAn answer that is clearly wrong.
Word fluffExtra wording that doesn't change the answer.

Explanation

graph TD
  Q[Read question] --> ASK[Find the real ask]
  ASK --> FLUFF[Remove word fluff]
  FLUFF --> QUAL[Identify the qualifier]
  QUAL --> ELIM[Eliminate impossible answers]
  ELIM --> MATCH[Match remaining to the qualifier]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class QUAL a;
  • Find the real ask — usually the final sentence (what must the solution achieve?).
  • Cut word fluff — long scenarios repeat irrelevant detail; ignore it.
  • Spot the qualifier — "most cost-effective", "least operational overhead", "highest performance", "most secure", "minimal changes". This chooses between answers that are all technically possible.
  • Eliminate self-eliminating (impossible/wrong) answers — often 2 drop away → then match the rest to the qualifier.

Exam Tip

Final Mock — Capstone

25 questions across the whole course: IAM, S3, VPC, EC2, Containers, RDS, DynamoDB, HA/Scaling, Serverless, CDN, Adv VPC, Hybrid, Security, CloudFormation, ML, Local Zones.
How to useApply the technique: find the ask, spot the qualifier, eliminate, then tap Show answer.
1. (IAM) Give an EC2 app permission to read S3 with no stored keys. Use?
A) IAM role on the instanceB) Access keys in codeC) Root userD) Bucket ACL
A. An instance role gives temporary credentials, no hardcoded keys.
2. (S3) Cheapest storage for rarely-accessed data needing 12h retrieval. Class?
A) S3 StandardB) Glacier Deep ArchiveC) S3 Standard-IAD) One Zone-IA
B. Deep Archive is cheapest for rare access with long retrieval times.
3. (VPC) Private subnet instances need outbound internet, no inbound. Use?
A) Internet GatewayB) NAT GatewayC) VPC peeringD) Egress-only IGW
B. NAT Gateway gives outbound-only IPv4 access.
4. (EC2) Lowest cost for a fault-tolerant batch job that can be interrupted. Use?
A) On-DemandB) Spot InstancesC) ReservedD) Dedicated Host
B. Spot is cheapest for interruptible, fault-tolerant work.
5. (HA) Distribute HTTP traffic across instances in multiple AZs. Use?
A) ALBB) NLBC) Route 53 onlyD) NAT
A. ALB is the Layer-7 load balancer for HTTP/HTTPS.
6. (Scaling) Keep average CPU at 50% automatically. Policy?
A) Target trackingB) ManualC) Scheduled onlyD) Step on a custom metric
A. Target tracking keeps a metric at a target value.
7. (RDS) Synchronous standby for automatic failover. Feature?
A) Read replicaB) Multi-AZC) SnapshotD) Aurora Serverless
B. Multi-AZ is a synchronous standby for HA failover (replicas scale reads).
8. (DynamoDB) Add a new query key to a live table. Use?
A) LSIB) GSIC) Recreate tableD) Scan
B. A GSI can be added anytime; LSI only at table creation.
9. (Serverless) Run code on an S3 upload event, no servers. Use?
A) Lambda (S3 trigger)B) EC2 cronC) BatchD) Step Functions only
A. S3 events trigger Lambda for serverless processing.
10. (Serverless) Decouple a producer + consumer with a buffer. Use?
A) SQSB) SNS onlyC) Kinesis FirehoseD) EFS
A. SQS is a pull-based queue that decouples + buffers.
11. (CDN) Lowest-latency global delivery of static + dynamic content + caching. Use?
A) CloudFrontB) Global AcceleratorC) S3 onlyD) Local Zones
A. CloudFront caches at edge; Global Accelerator routes but doesn't cache.
12. (CDN) Lock an S3 origin so only CloudFront can read it. Use?
A) OAC + bucket policyB) Public bucketC) Presigned URL onlyD) NACL
A. Origin Access Control + a bucket policy restricts access to CloudFront.
13. (Adv VPC) Reach S3 from a private subnet free + private. Use?
A) Gateway endpointB) Interface endpointC) NATD) IGW
A. S3 (and DynamoDB) use a free gateway VPC endpoint.
14. (Adv VPC) Connect 30 VPCs + on-prem with transitive routing. Use?
A) Peering meshB) Transit GatewayC) NATD) IGW
B. Transit Gateway gives transitive routing and replaces a mesh.
15. (Hybrid) Encrypted on-prem↔AWS link working today. Use?
A) Direct ConnectB) Site-to-Site VPNC) PeeringD) Snowball
B. VPN is quick + encrypted; DX takes weeks to provision.
16. (Hybrid) Move 80 TB from a slow-link site + run on-site compute. Use?
A) SnowconeB) Snowball EdgeC) DataSyncD) Snowmobile
B. Snowball Edge handles ~80 TB + edge compute.
17. (Security) Find sensitive PII inside S3 objects. Use?
A) MacieB) InspectorC) GuardDutyD) Config
A. Macie discovers PII in S3 (Inspector=vulns, GuardDuty=threats).
18. (Security) FIPS 140-2 L3 with keys AWS cannot access. Use?
A) KMSB) CloudHSMC) Secrets ManagerD) Config
B. CloudHSM is single-tenant, FIPS 140-2 L3, keys only you control.
19. (Security) Block SQL-injection in HTTPS to an ALB. Use?
A) WAF Web ACLB) NACLC) Security groupD) Shield Standard
A. WAF inspects L7 content; SG/NACL only see IP/port.
20. (CloudFormation) Preview how a stack change affects resources before applying. Use?
A) Change SetsB) DependsOnC) Drift onlyD) Delete + recreate
A. Change Sets preview proposed changes before execution.
21. (CloudFormation) Keep an S3 bucket when its stack is deleted. Use?
A) DeletionPolicy: RetainB) DependsOnC) ConditionD) Nested stack
A. DeletionPolicy: Retain preserves the resource on stack delete.
22. (ML) Convert recorded calls to text, then score sentiment. Combo?
A) Transcribe + ComprehendB) Polly + TranslateC) Lex + KendraD) Rekognition + Textract
A. Transcribe makes text; Comprehend scores sentiment.
23. (ML) Build + train + host a custom model. Use?
A) SageMakerB) ComprehendC) RekognitionD) Forecast
A. SageMaker is for custom models; the rest are pre-built.
24. (Local Zones) Ultra-low latency for users in one city, on AWS infra. Use?
A) Local ZonesB) OutpostsC) CloudFront onlyD) Region in another continent
A. Local Zones extend a Region into a metro for low latency.
25. (Technique) Two answers both work; the ask says "least operational overhead". Pick?
A) Self-managed EC2B) Managed/serverless optionC) Cheapest regardlessD) Most powerful
B. "Least operational overhead" points to managed/serverless services.
