VPC Flow Logs

See who talks to whom (metadata only).

Quick Understanding

ConceptMeaningExam relevance
Metadata onlyCaptures src/dst, ports, ACCEPT/REJECT.Not packet contents.
3 levelsVPC, Subnet, or Network Interface.Where you attach it.
DestinationsS3 or CloudWatch Logs.Where logs go.
Not real-timeAggregated (1 or 10 min).Slight delay.

Key Terms

TermSimple meaning
ENIElastic Network Interface — a virtual network card.
Packet snifferA tool that reads the actual data inside packets.
ACCEPT / REJECTWhether security rules allowed or blocked the traffic.

Explanation

graph TD
  LVL{Attach at} --> V[VPC: all ENIs]
  LVL --> S[Subnet: ENIs in subnet]
  LVL --> E[ENI: one interface]
  V --> DEST[Capture metadata]
  S --> DEST
  E --> DEST
  DEST --> S3[S3]
  DEST --> CW[CloudWatch Logs]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class DEST a;
  • Flow Logs capture metadata about IP traffic (source/destination IP, ports, protocol, bytes, ACCEPT/REJECT) — not the packet contents (that needs a packet sniffer / Traffic Mirroring).
  • Attach at VPC, Subnet, or ENI level; filter ACCEPT, REJECT, or ALL.
  • Send logs to S3 or CloudWatch Logs. It's not real-time (aggregated, 1 or 10 min).
  • Some traffic is not captured: Amazon DNS, instance metadata (169.254.169.254), DHCP, Windows licence. [added for exam]

Demo

Demo · CLI v2 (logs are tiny; CW storage only)Situation: log all traffic in the default VPC to CloudWatch. Check first: CLI configured.
Create a log group + a flow-logs role
VPC=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
aws logs create-log-group --log-group-name a4l-flowlogs
ROLE=$(aws iam create-role --role-name a4l-flowlogs-role \
  --assume-role-policy-document '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Principal":{"Service":"vpc-flow-logs.amazonaws.com"},"Action":"sts:AssumeRole"}]}' \
  --query Role.Arn --output text)
aws iam put-role-policy --role-name a4l-flowlogs-role --policy-name cwlogs \
  --policy-document '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":["logs:CreateLogStream","logs:PutLogEvents","logs:DescribeLogStreams"],"Resource":"*"}]}'
sleep 10
Expect: a role ARN; wait ~10s for IAM to propagate.
Create the flow log (capture ALL)
aws ec2 create-flow-logs --resource-type VPC --resource-ids $VPC \
  --traffic-type ALL --log-group-name a4l-flowlogs \
  --deliver-logs-permission-arn $ROLE
Expect: a FlowLogId. Records appear in the log group after a few minutes (showing ACCEPT/REJECT lines).
Fix / Key infoFlow Logs show metadata only. To see packet contents you need Traffic Mirroring / a packet sniffer.
Clean up
FL=$(aws ec2 describe-flow-logs --filter Name=resource-id,Values=$VPC \
  --query "FlowLogs[0].FlowLogId" --output text)
aws ec2 delete-flow-logs --flow-log-ids $FL
aws iam delete-role-policy --role-name a4l-flowlogs-role --policy-name cwlogs
aws iam delete-role --role-name a4l-flowlogs-role
aws logs delete-log-group --log-group-name a4l-flowlogs
Expect: flow log, role + log group removed.

Egress-Only Internet Gateway

Outbound-only internet for IPv6.

Quick Understanding

ConceptMeaningExam relevance
IPv6 onlyFor IPv6-enabled instances.Not for IPv4.
Outbound + returnAllows out (and replies), blocks inbound-initiated.Stateful.
Why neededIPv6 has no NAT (all IPv6 is public).Key reasoning.

Key Terms

TermSimple meaning
EgressOutbound (leaving the VPC).
NAT GatewayOutbound-only internet for private IPv4.
StatefulReturn traffic is automatically allowed.

Explanation

graph TD
  V4[Private IPv4 instance] -->|outbound only| NAT[NAT Gateway]
  V6[IPv6 instance] -->|outbound only| EIGW[Egress-Only IGW]
  NAT --> NET[Internet]
  EIGW --> NET
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class EIGW a;
  • IPv6 addresses are all public/globally routable, so there is no NAT for IPv6.
  • An Egress-Only Internet Gateway gives IPv6 instances outbound-only internet (and allows return traffic) while blocking inbound-initiated connections.
  • Compare: IPv4 private → NAT Gateway; IPv6 → Egress-Only IGW.

Demo

Demo · CLI v2 (free to create)Situation: create an egress-only IGW for IPv6. Check first: CLI configured.
Create the egress-only IGW
VPC=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
EIGW=$(aws ec2 create-egress-only-internet-gateway --vpc-id $VPC \
  --query "EgressOnlyInternetGateway.EgressOnlyInternetGatewayId" --output text)
echo $EIGW
Expect: an ID like eigw-0.... In a real IPv6 setup you'd add a route ::/0 → eigw in the subnet's route table.
Fix / Key infoIt only works for IPv6. For private IPv4 outbound you still use a NAT Gateway.
Clean up
aws ec2 delete-egress-only-internet-gateway \
  --egress-only-internet-gateway-id $EIGW
Expect: egress-only IGW removed.

VPC Endpoints (Gateway)

Private S3 & DynamoDB, no internet.

Quick Understanding

ConceptMeaningExam relevance
S3 & DynamoDB onlyGateway endpoints serve just these two.Remember the pair.
Route table entryAdds a prefix list; router sends traffic privately.How it works.
Free + HANo cost; resilient across AZs.Cheap private access.
Regional onlyCan't be used cross-region or from outside.Limitation.

Key Terms

TermSimple meaning
Prefix listA named set of IP ranges for a service.
Endpoint policyControls what the endpoint may access.
Private subnetA subnet with no internet route.

Explanation

graph TD
  EC2[Private EC2] --> RT[Route table - prefix list to S3]
  RT --> GWEP[Gateway endpoint]
  GWEP --> S3[S3 / DynamoDB - private, no internet]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class GWEP a;
  • Gateway endpoints give private access to S3 and DynamoDB only — no internet gateway or NAT needed.
  • They add a prefix list to the route table, so the VPC router sends that traffic to the endpoint.
  • They are free, highly available (route-table target, not an ENI), and regional (no cross-region or external access).
  • An endpoint policy can restrict which buckets/tables are reachable.

Demo

Demo · CLI v2 (free)Situation: let a private subnet reach S3 privately. Check first: CLI configured.
Create an S3 gateway endpoint
VPC=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
RT=$(aws ec2 describe-route-tables --filters Name=vpc-id,Values=$VPC \
  --query "RouteTables[0].RouteTableId" --output text)
REGION=$(aws configure get region)
EP=$(aws ec2 create-vpc-endpoint --vpc-id $VPC --vpc-endpoint-type Gateway \
  --service-name com.amazonaws.$REGION.s3 --route-table-ids $RT \
  --query "VpcEndpoint.VpcEndpointId" --output text)
echo $EP
Expect: an endpoint ID + a new prefix-list route in the route table. S3 traffic now stays private.
Fix / Key infoOnly S3 + DynamoDB use gateway endpoints. Everything else uses interface endpoints (L4).
Clean up
aws ec2 delete-vpc-endpoints --vpc-endpoint-ids $EP
Expect: endpoint removed + route cleaned up.

Connects to NOW

Needed LATER

Exam Pointers

Exam Tip
  • Gateway endpoint = S3 + DynamoDB only, free, route-table based, HA.
  • Can't be reached from on-prem / other VPC / other region.

Flashcards

VPC Endpoints (Interface)

Private access to most other services.

Quick Understanding

ConceptMeaningExam relevance
Other servicesEverything except S3/DynamoDB gateway.Uses PrivateLink.
It's an ENIA private IP network card in a subnet.SG controls it.
Not HA by defaultOne ENI per AZ for HA.Common trap.
DNS-drivenPrivate DNS resolves service name to the ENI.How apps use it.

Key Terms

TermSimple meaning
PrivateLinkThe tech behind interface endpoints (private service access).
Private DNSMakes the normal service URL resolve to the private ENI.
Security groupFirewall controlling access to the endpoint ENI.

Explanation

graph TD
  EC2[Private EC2] --> ENI[Interface endpoint ENI - private IP, SG]
  ENI -->|PrivateLink| SVC[AWS service e.g. EC2/SSM/SNS]
  ENI --> DNS[Private DNS resolves service name]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class ENI a;
  • Interface endpoints give private access to most other AWS services (not the S3/DynamoDB gateway pair) using PrivateLink.
  • An interface endpoint is an ENI (private IP) in a subnet, controlled by a security group.
  • It is not HA by default — it's a single ENI; place one per AZ for full HA.
  • Apps reach it via the endpoint's DNS name, or enable private DNS so the normal service URL resolves to the ENI.

Demo

Demo · CLI v2 (small hourly cost — delete promptly)Situation: create an interface endpoint for the EC2 API. Check first: a subnet ID + a security group ID in the default VPC.
Create an interface endpoint
VPC=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
SUBNET=$(aws ec2 describe-subnets --filters Name=vpc-id,Values=$VPC \
  --query "Subnets[0].SubnetId" --output text)
SG=$(aws ec2 describe-security-groups --filters Name=vpc-id,Values=$VPC \
  --query "SecurityGroups[0].GroupId" --output text)
REGION=$(aws configure get region)
EP=$(aws ec2 create-vpc-endpoint --vpc-id $VPC --vpc-endpoint-type Interface \
  --service-name com.amazonaws.$REGION.ec2 \
  --subnet-ids $SUBNET --security-group-ids $SG \
  --query "VpcEndpoint.VpcEndpointId" --output text)
echo $EP
Expect: an endpoint ID + an ENI with a private IP. Enable private DNS so ec2.<region>.amazonaws.com resolves privately.
Cost warningInterface endpoints bill per hour + per GB. Delete it right after testing.
Clean up
aws ec2 delete-vpc-endpoints --vpc-endpoint-ids $EP
Expect: endpoint + its ENI removed (stops billing).

VPC Peering

Link two VPCs privately.

Quick Understanding

ConceptMeaningExam relevance
1:1 linkConnect exactly two VPCs.Same/diff account/region.
Not transitiveA–B + B–C does NOT give A–C.Classic trap.
No overlapCIDRs must not overlap.Hard requirement.
Needs routesAdd routes both sides; SG refs same-region only.Setup detail.

Key Terms

TermSimple meaning
Peering connectionThe logical link between two VPCs.
Transitive"Friend of a friend" routing (peering is NOT this).
CIDRThe VPC's IP address range.

Explanation

graph LR
  A[VPC A] --- B[VPC B]
  B --- C[VPC C]
  A -. no direct path .- C
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class B a;
  • VPC peering is a logical 1:1 private link between two VPCs — same or different account/region; cross-region traffic uses the AWS network (encrypted).
  • It is NOT transitive: if A↔B and B↔C, A still cannot reach C — you need a direct A↔C peer.
  • CIDRs must not overlap. You must add route table entries on both sides; you can reference a peer's security group by ID only in the same region.
  • For many VPCs at scale, use Transit Gateway instead. [added for exam]

Demo

Demo · CLI v2 (peering is free)Situation: peer the default VPC with a new test VPC. Check first: CLI configured. Use a non-overlapping CIDR.
Make a second VPC + request peering
VPC1=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
VPC2=$(aws ec2 create-vpc --cidr-block 10.99.0.0/16 \
  --query "Vpc.VpcId" --output text)
PCX=$(aws ec2 create-vpc-peering-connection --vpc-id $VPC1 \
  --peer-vpc-id $VPC2 \
  --query "VpcPeeringConnection.VpcPeeringConnectionId" --output text)
echo $PCX
Expect: a peering ID in pending-acceptance.
Accept the peering
aws ec2 accept-vpc-peering-connection --vpc-peering-connection-id $PCX \
  --query "VpcPeeringConnection.Status.Code"
Expect: active. To actually route, add routes on both VPCs pointing the other's CIDR at $PCX.
Fix / Key infoPeering needs routes on both sides and non-overlapping CIDRs; it is not transitive.
Clean up
aws ec2 delete-vpc-peering-connection --vpc-peering-connection-id $PCX
aws ec2 delete-vpc --vpc-id $VPC2
Expect: peering + test VPC removed.
