Architecture Deep Dive

From monolith to event-driven.

Quick Understanding

StyleMeaningExam relevance
MonolithEverything in one unit.Scales all-or-nothing; single failure point.
TieredSplit web / app / data.Each tier scales separately.
Queue-basedDecouple tiers with a queue.Async + worker pools.
MicroservicesMany small services.Independent deploy/scale.
Event-drivenReact to events; no idle wait.Serverless-friendly.

Key Terms

TermSimple meaning
DecoupleLet parts work without depending directly on each other.
AsynchronousSend work and move on; reply comes later.
Worker poolA set of workers pulling tasks from a queue.
EventA signal that something happened.

Explanation

graph LR
  M[Monolith] --> T[Tiered]
  T --> Q[Queue-based decoupled]
  Q --> MS[Microservices]
  MS --> ED[Event-driven]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class ED a;
  • A monolith puts everything together — simple but it scales and fails as one block.
  • Tiered splits web/app/data so each can scale; a queue between tiers adds async decoupling + worker pools.
  • Microservices are many small independent services; event-driven designs have services react to events with no idle waiting — ideal for serverless.

AWS Lambda

Run code without servers (FaaS).

Quick Understanding

ConceptMeaningExam relevance
FaaSYou upload a function; AWS runs it on demand.Pay per request + duration.
LimitsMax 15-min timeout; memory 128MB–10GB.Long jobs → Step Functions.
InvocationSync, async, or poll (event source mapping).Behaviour differs.
Versions/aliasesImmutable versions + movable aliases.Safe deploys.

Key Terms

TermSimple meaning
RuntimeThe language environment (e.g. Python 3.9).
Execution roleThe IAM role giving the function AWS permissions.
Cold startExtra delay when a new environment spins up.
Event source mappingLambda polls a source (SQS/Kinesis/DynamoDB streams).

Explanation

graph TD
  TRIG[Trigger: API GW / S3 / SQS / schedule] --> FN[Lambda function + execution role]
  FN --> LOG[CloudWatch Logs]
  FN --> SYNC[Sync: caller waits]
  FN --> ASYNC[Async: retries 2x → DLQ/destinations]
  FN --> POLL[Poll: SQS/Kinesis/DynamoDB streams]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class FN a;
  • Lambda runs a function in a runtime (e.g. Python 3.9) using an execution role; you pay per request + duration (GB-seconds).
  • Limits: 15-minute max timeout, memory 128MB–10GB (CPU scales with memory), /tmp ephemeral storage. Long workflows → Step Functions (L6).
  • Networking: by default Lambda has internet access; attach it to your VPC to reach private resources (then it needs a NAT for internet).
  • Invocation: synchronous (caller waits, e.g. API GW), asynchronous (events, retries twice → DLQ/destinations), event source mapping (polls SQS/Kinesis/DynamoDB streams).
  • Versions are immutable; aliases point to a version for safe rollouts.

Demo

Demo · CLI v2 (free tier is generous)Situation: create + invoke a Python Lambda. Check first: CLI configured.
Make the function + an execution role
echo 'def handler(event, context):
    return {"message": "hello from lambda"}' > fn.py
zip fn.zip fn.py
ROLE=$(aws iam create-role --role-name a4l-lambda-role \
  --assume-role-policy-document '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Principal":{"Service":"lambda.amazonaws.com"},"Action":"sts:AssumeRole"}]}' \
  --query Role.Arn --output text)
aws iam attach-role-policy --role-name a4l-lambda-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
sleep 10
Expect: a role ARN; wait ~10s so IAM propagates.
Create the function
aws lambda create-function --function-name a4l-hello \
  --runtime python3.9 --handler fn.handler \
  --zip-file fileb://fn.zip --role $ROLE
Expect: function details with State: Active.
Invoke it
aws lambda invoke --function-name a4l-hello out.json
cat out.json
Expect: {"message": "hello from lambda"}.
Fix / Key infoThe course uses Python 3.9; newer runtimes exist (verify what's current). Logs go to CloudWatch Logs via the basic execution role.
Clean up
aws lambda delete-function --function-name a4l-hello
aws iam detach-role-policy --role-name a4l-lambda-role \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole
aws iam delete-role --role-name a4l-lambda-role
rm -f fn.py fn.zip out.json
Expect: function + role removed.

Connects to NOW
  • Backend for API GW (L7), targets for SNS/EventBridge, tasks in Step Functions.
Needed LATER
  • Almost every serverless pattern.

Exam Tip

  • Give permissions via the execution role, not keys.
  • Put Lambda in a VPC to reach private resources (needs NAT for internet).
  • Async invokes retry twice and can send failures to DLQ/destinations.

Explanation

graph TD
  SRC[AWS service events] --> BUS[Event bus]
  SCHED[Schedule cron/rate] --> RULE[Rule]
  BUS --> RULE

Simple Notification Service (SNS)

Pub/sub topics with fan-out.

Quick Understanding

ConceptMeaningExam relevance
Pub/subPublishers send to a topic; subscribers receive.Push messaging.
SubscribersLambda, SQS, HTTP/S, email, SMS, mobile push.Many target types.
Fan-outOne message → many SQS queues.Classic pattern.
Filter policySubscribers get only matching messages.Selective delivery.

Key Terms

TermSimple meaning
TopicA named channel messages are sent to.
Publisher / SubscriberSender / receiver of topic messages.
Fan-outDeliver one message to many endpoints at once.

Explanation

graph TD
  PUB[Publisher] --> TOPIC[SNS Topic]
  TOPIC --> S1[Lambda]
  TOPIC --> S2[SQS queue]
  TOPIC --> S3[Email / SMS / HTTP]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class TOPIC a;
  • Publishers send messages to a topic; all subscribers of that topic receive them (push).
  • Subscriber types include Lambda, SQS, HTTP/S, email, SMS, mobile push.
  • Fan-out: publish once, deliver to many SQS queues. Filter policies send only matching messages. FIFO topics add ordering (to FIFO SQS).

Demo

Demo · CLI v2 (free-ish)Situation: create a topic, subscribe email, publish. Check first: CLI configured + an email you can confirm.
Create the topic + subscribe your email
TOPIC=$(aws sns create-topic --name a4l-topic --query TopicArn --output text)
aws sns subscribe --topic-arn $TOPIC --protocol email \
  --notification-endpoint you@example.com
Expect: a topic ARN + a confirmation email — click the link to confirm the subscription.
Publish a message
aws sns publish --topic-arn $TOPIC --subject "A4L" --message "hello from SNS"
Expect: a message ID; the email arrives (once confirmed).
Fix / Key infoEmail subscriptions must be confirmed before delivery. For fan-out, subscribe several SQS queues to one topic.
Clean up
aws sns delete-topic --topic-arn $TOPIC
Expect: topic + subscriptions removed.

Exam Pointers

Exam Tip
  • SNS = push pub/sub; SQS = pull queue. Fan-out = SNS topic → many SQS queues.
  • Email subs need confirmation; use filter policies for selective delivery.

Quick Understanding

ConceptMeaningExam relevance
State machineA workflow defined in JSON (ASL).Orchestration.
StatesTask, Choice, Parallel, Map, Wait…Building blocks.
Standard vs ExpressLong/exactly-once vs high-volume/short.Pick by workload.
Beats long LambdaHandles >15-min, multi-step flows.Lambda limit fix.

Key Terms

TermSimple meaning
ASLAmazon States Language — the JSON workflow format.
Task stateDoes work (e.g. calls a Lambda).
Choice stateBranches based on a condition.

Explanation

graph TD
  START[Start] --> T1[Task: Lambda]
  T1 --> C{Choice}
  C -->|yes| T2[Task]
  C -->|no| W[Wait]
  T2 --> END[End]
  W --> END
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class C a;
  • Step Functions runs a state machine (workflow) written in ASL JSON, with states like Task, Choice, Parallel, Map, Wait.
  • Standard workflows run up to 1 year (exactly-once, default); Express are for high-volume, short (<5 min) runs.
  • It orchestrates long, multi-step processes that exceed Lambda's 15-minute limit.

Demo

Demo · CLI v2 (a Pass-only machine runs nearly free)Situation: create a tiny state machine. Check first: an IAM role for Step Functions ($SFROLE ARN).
Define + create the state machine

"States":{"Hello":{"Type":"Pass","Result":"Hello A4L","End":true}}}' > sm.json

  --definition file://sm.json --role-arn $SFROLE \
  --query stateMachineArn --output text
Expect: a state-machine ARN. Start an execution with start-execution and it finishes instantly (Pass state).
Fix / Key infoReal workflows use Task states calling Lambda. The role must allow Step Functions to invoke those services.
Clean up
SM=$(aws stepfunctions list-state-machines \
  --query "stateMachines[?name=='a4l-sm'].stateMachineArn" --output text)

Expect: state machine removed.

Demo · CLI v2 (free tier covers light use)Situation: create a tiny REST API with a MOCK response. Check first: CLI configured.

Add a MOCK integration + deploy
aws apigateway put-integration --rest-api-id $API --resource-id $ROOT \
  --http-method GET --type MOCK \

aws apigateway put-method-response --rest-api-id $API --resource-id $ROOT \
  --http-method GET --status-code 200
aws apigateway put-integration-response --rest-api-id $API --resource-id $ROOT \
  --http-method GET --status-code 200 \

Expect: a deployment. Test: https://$API.execute-api.<region>.amazonaws.com/prod returns the JSON.

Quick Understanding

ConceptMeaningExam relevance
Managed queueStore messages between components.Decoupling.
Pull-basedWorkers poll for messages.Worker pools.
Visibility timeoutMessage hidden after receive until deleted.Default 30s.
Scale on lengthASG can scale on queue depth.Elastic workers.

Key Terms

TermSimple meaning
Visibility timeoutTime a received message stays hidden from others.
PollingAsking the queue for messages (short or long).
RetentionHow long unread messages are kept (max 14 days).

Explanation

graph TD
  PROD[Producer] --> Q[SQS queue]
  Q --> W1[Worker 1 polls]
  Q --> W2[Worker 2 polls]
  W1 --> DEL[Process then DELETE message]
