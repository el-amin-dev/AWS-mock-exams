BGP 101

How big networks share routes (advanced/optional).

Quick Understanding

ConceptMeaningExam relevance
BGPThe internet's routing protocol.Used by DX + dynamic VPN.
ASNA number that names a network.Identifies each side.
AS_PATHList of networks a route crosses.Shorter is preferred.
Dynamic routingRouters learn paths automatically.Auto-failover.

Key Terms

TermSimple meaning
ASNAutonomous System Number — an ID for a network.
Path-vectorRouting based on the list of networks to cross.

Explanation

  • BGP is how separate networks exchange routes; each network has an ASN.
  • It is path-vector: routes carry an AS_PATH (the networks crossed); a shorter path is preferred.
  • AWS uses BGP for Direct Connect and dynamic (BGP) Site-to-Site VPN to learn routes automatically + fail over.
  • This is an advanced/optional topic — for SAA you mainly need "BGP = dynamic routing for DX/VPN".

IPSec VPN Fundamentals

Encrypted tunnels over the public internet.

Quick Understanding

ConceptMeaningExam relevance
IPsecEncrypts traffic across an untrusted network.VPN basis.
Two phasesPhase 1 (IKE) then Phase 2 (IPsec SA).Classic exam point.
Interesting trafficTraffic matching a policy triggers the tunnel.When it activates.
Policy vs route basedMatch traffic vs use route tables.Route-based = dynamic.

Key Terms

TermSimple meaning
SASecurity Association — agreed keys/settings for a tunnel.
IKEThe process that sets up the secure channel (Phase 1).
Symmetric / asymmetricSame key both sides / key pair.

Explanation

graph TD
  P1[Phase 1 - IKE] --> CH[Build secure channel - slow, asymmetric]
  CH --> P2[Phase 2 - IPsec SA]
  P2 --> TUN[Encrypted tunnel - fast, symmetric]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class TUN a;
  • IPsec creates encrypted tunnels over an untrusted network (the internet).
  • Phase 1 (IKE) builds a secure channel (slow, uses asymmetric keys + Diffie-Hellman) → an IKE SA.
  • Phase 2 negotiates the IPsec SA for the actual traffic (faster, symmetric keys).
  • Interesting traffic (matching a policy) triggers a tunnel. Route-based VPNs (route tables) support dynamic routing; policy-based match specific traffic.

AWS Site-to-Site VPN

Managed encrypted link to on-premises.

Quick Understanding

ConceptMeaningExam relevance
VGWVirtual Private Gateway (AWS side).Or attach to TGW.
CGWCustomer Gateway (your router).On-prem side.
2 tunnelsTwo endpoints for HA.Resilience.
vs Direct ConnectQuick + encrypted, but over internet.Performance trade-off.

Key Terms

TermSimple meaning
VGWThe AWS-side VPN endpoint attached to a VPC.
CGWA logical record of your on-prem router (its IP + ASN).
TunnelOne encrypted IPsec connection (you get two).

Explanation

graph LR
  ONP[On-prem router = CGW] -->|2 IPsec tunnels over internet| VGW[VGW on VPC]
  VGW --> VPC[VPC resources]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class VGW a;
  • A managed IPsec VPN between AWS and on-prem: VGW (AWS side, or attach to a TGW) + CGW (your router) + a VPN connection with 2 tunnels for HA.
  • Routing is static (manual routes) or dynamic (BGP).
  • It's quick to set up (hours), encrypted, ~1.25 Gbps per tunnel, but runs over the public internet (variable latency).
  • vs Direct Connect: VPN = fast to deploy + encrypted but lower/variable performance; DX = slow to provision, high consistent performance.

Demo

Demo · the Site-to-Site VPN mini-projectSituation: connect AWS to a simulated on-prem site (pfSense router). The full lab is a 1-click CloudFormation deploy (needs a us-east-1 SSH key + a free pfSense Marketplace subscription). Below is the AWS side you can build with the CLI.
Cost warningA VPN connection bills per hour while it exists (the VGW + CGW are free). Delete the VPN connection right after testing.
Create the VGW + attach it; create the CGW
VPC=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
VGW=$(aws ec2 create-vpn-gateway --type ipsec.1 \
  --query "VpnGateway.VpnGatewayId" --output text)
aws ec2 attach-vpn-gateway --vpn-gateway-id $VGW --vpc-id $VPC
CGW=$(aws ec2 create-customer-gateway --type ipsec.1 \
  --public-ip 203.0.113.10 --bgp-asn 65000 \
  --query "CustomerGateway.CustomerGatewayId" --output text)
echo "$VGW $CGW"
Expect: a VGW + CGW ID (free). Use your real on-prem router's public IP in production.
Create the VPN connection (BILLS — delete after)
VPN=$(aws ec2 create-vpn-connection --type ipsec.1 \
  --customer-gateway-id $CGW --vpn-gateway-id $VGW \
  --options StaticRoutesOnly=true \
  --query "VpnConnection.VpnConnectionId" --output text)
echo $VPN
Expect: a VPN connection ID + 2 tunnel endpoints. The download config holds the settings for your router.
Fix / Key infoYou always get 2 tunnels for HA. Use StaticRoutesOnly=false for BGP dynamic routing.
Clean up
aws ec2 delete-vpn-connection --vpn-connection-id $VPN
aws ec2 detach-vpn-gateway --vpn-gateway-id $VGW --vpc-id $VPC
aws ec2 delete-vpn-gateway --vpn-gateway-id $VGW
aws ec2 delete-customer-gateway --customer-gateway-id $CGW
Expect: all removed (delete the VPN connection first to stop billing).

  • VPN is also a common backup for Direct Connect.

graph TD

  • Transit Gateway is a hub that connects many VPCs, VPNs and Direct Connect (via Transit VIF) with transitive routing.
  • Unlike peering (1:1, non-transitive), one TGW replaces a complex mesh.
  • Works cross-account (shared via RAM) and cross-region (TGW peering).

  --query "TransitGateways[0].State"
Expect: a TGW ID; state goes pending → available. You'd then add VPC/VPN attachments.
Fix / Key infoTGW gives transitive routing (peering does not). For many VPCs it's far simpler than a peering mesh.
