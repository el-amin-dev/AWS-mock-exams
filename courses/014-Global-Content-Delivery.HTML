CloudFront Architecture

AWS's content delivery network (CDN).

Quick Understanding

ConceptMeaningExam relevance
CDNCache content near users to cut latency.Faster delivery.
DistributionThe CloudFront config unit.What you create.
OriginWhere the real content lives (S3 or HTTP).Source of truth.
Edge locationCache site close to users (400+).Serves cached copies.

Key Terms

TermSimple meaning
Edge locationA small AWS site near users that caches content.
Regional edge cacheA bigger mid-tier cache between edge and origin.
Cache hit / missFound in cache / had to fetch from origin.
ViewerThe user/browser requesting content.

Explanation

graph TD
  V[Viewer] --> EDGE[Nearest edge location]
  EDGE -->|cache miss| REC[Regional edge cache]
  REC -->|cache miss| ORI[Origin: S3 or HTTP server]
  EDGE -->|cache hit| V
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class EDGE a;
  • A viewer connects to the nearest edge location. On a cache hit it's served instantly; on a miss CloudFront fetches from the regional edge cache, then the origin.
  • You create a distribution pointing at one or more origins (S3 bucket or any HTTP server).
  • Caching near users lowers latency and reduces load on the origin.

Key Terms

Explanation

  • Distribution-level settings apply to everything: alternate domain names (CNAMEs), SSL certificate, WAF, price class, default root object.

  • The default behaviour (*) catches anything not matched by a more specific pattern.

TTL & Invalidations

How long objects stay cached, and how to clear them.

Quick Understanding

ConceptMeaningExam relevance
TTLHow long an object is cached at the edge.Default 24h.
Min / Max TTLBounds on cache time.Limit origin headers.
Cache-ControlOrigin header setting cache time.Per-object control.
InvalidationForce-remove objects from cache.Costs after free tier.

Key Terms

TermSimple meaning
TTLTime To Live — how long a cached copy is valid.
InvalidationTelling CloudFront to drop cached copies now.
Versioned filenameNew file name per change (e.g. logo.v2.png).

Explanation

  • TTL sets how long an object stays cached before the edge re-checks the origin. Default TTL is 24 hours; Min/Max TTL bound what origin headers can set.
  • The origin can control caching with Cache-Control: max-age / Expires headers, kept within Min/Max TTL.
  • Invalidation force-removes objects (e.g. /*) — useful but costs money after 1000 paths/month and takes time.
  • Better practice: use versioned filenames so new content has a new URL — no waiting, no invalidation cost. [added for exam]

Connects to NOW

Needed LATER
  • Cache strategy for any CDN-fronted app.

Exam Pointers

Exam Tip
  • Default TTL = 24 hours; origin Cache-Control is bounded by Min/Max TTL.
  • Update content fast + cheap → versioned filenames, not invalidations.

ACM (Certificate Manager)

Free SSL/TLS certificates for AWS services.

Quick Understanding

ConceptMeaningExam relevance
Manages certsCreate, store, auto-renew SSL/TLS certs.No manual renewal.
Public = freePublic certs cost nothing.Private CA costs.
IntegrationsDeploy to CloudFront, ALB, API Gateway.Not EC2 directly.
Region ruleCloudFront needs the cert in us-east-1.Classic trap.

Key Terms

TermSimple meaning
CertificateProves a site's identity + enables HTTPS.
DNS validationProve you own the domain via a DNS record.
Auto-renewalACM renews its public certs automatically.

Explanation

graph TD
  ACM[ACM: request + auto-renew] --> CF[CloudFront - cert in us-east-1]
  ACM --> ALB[ALB - cert in its region]
  ACM --> API[API Gateway]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class ACM a;
  • ACM creates, stores and auto-renews SSL/TLS certificates. Public certs are free; private certs (Private CA) cost money.
  • Deploy certs to integrated services: CloudFront, ALB, API Gateway — you cannot install an ACM cert directly on EC2.
  • ACM is regional: a cert must be in the resource's region. CloudFront requires the cert in us-east-1 (N. Virginia).

Demo

Demo · CLI v2 (public certs are free)Situation: request a public cert for CloudFront. Check first: use a domain you own; create it in us-east-1.
Request a DNS-validated cert in us-east-1
CERT=$(aws acm request-certificate --domain-name www.example.com \
  --validation-method DNS --region us-east-1 \
  --query CertificateArn --output text)
aws acm describe-certificate --certificate-arn $CERT --region us-east-1 \
  --query "Certificate.Status"
Expect: a cert ARN + status PENDING_VALIDATION. Add the shown CNAME to your DNS (Route 53) to validate; then it becomes ISSUED.
Fix / Key infoFor CloudFront the cert must be in us-east-1, no matter where your origin is. ACM auto-renews issued public certs.
Clean up
aws acm delete-certificate --certificate-arn $CERT --region us-east-1
Expect: cert removed (detach from any service first).

CloudFront & SSL/TLS

Viewer side, origin side, and SNI.

Quick Understanding

ConceptMeaningExam relevance
Viewer sideViewer ↔ CloudFront encryption.Cert in us-east-1.
Origin sideCloudFront ↔ origin encryption.Origin needs valid cert.
SNIShare one IP across many SSL sites.Free, modern clients.
Dedicated IP SSLFor very old clients.Legacy + expensive.

Key Terms

TermSimple meaning
SNIServer Name Indication — says which site you want during the SSL handshake.
Viewer / origin protocolHow each side connects (HTTP/HTTPS).

Explanation

graph LR
  V[Viewer] -->|HTTPS - viewer cert in us-east-1| CF[CloudFront]
  CF -->|HTTPS - origin needs valid cert| ORI[Origin]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class CF a;
  • There are two SSL connections: viewer side (viewer ↔ CloudFront, cert from ACM in us-east-1) and origin side (CloudFront ↔ origin, the origin needs a valid public cert; S3 supports HTTPS).
  • SNI lets many HTTPS sites share one IP by naming the site in the handshake — it's free and works on modern clients.
  • Dedicated IP SSL is only for very old clients that don't support SNI — it's expensive (legacy).

Quick Understanding

ConceptMeaningExam relevance

Custom originAny HTTP server (EC2/ALB/on-prem).Dynamic content.
S3 website endpointTreated as a custom origin.Enables redirects/index docs.

Key Terms

TermSimple meaning
S3 origin (REST)Bucket accessed via its S3 API endpoint.
Custom originAny web server CloudFront fetches from over HTTP(S).
Default root objectFile served for "/" (e.g. index.html).

Explanation

graph TD
  CF[CloudFront distribution] --> S3[S3 origin - static, lock with OAC]
  CF --> CUST[Custom origin - EC2/ALB/on-prem/S3 website]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class CF a;
  • S3 origin (REST endpoint) is for static files — secure it so only CloudFront can read it (OAC, L8).
  • Custom origin is any HTTP server (EC2, ALB, on-prem). The S3 static website endpoint counts as a custom origin (enables index docs + redirects).
  • Per-origin settings: protocol, origin path, custom headers.

Demo

Demo · CLI v2 (CloudFront has a generous free tier)Situation: put a CloudFront CDN in front of an S3 bucket. Check first: CLI configured. (Distribution is free to create; you pay only on traffic.)
Create the bucket + upload a page
BUCKET=a4l-cf-$RANDOM
aws s3 mb s3://$BUCKET
echo '
Hello from A4L via CloudFront
' > index.html
aws s3 cp index.html s3://$BUCKET/
Expect: the bucket is made + the file uploads.
Create a distribution with the S3 origin
DIST=$(aws cloudfront create-distribution \
  --origin-domain-name $BUCKET.s3.amazonaws.com \
  --default-root-object index.html \
  --query "Distribution.Id" --output text)
aws cloudfront get-distribution --id $DIST \
  --query "Distribution.DomainName" --output text
Expect: a distribution ID + a *.cloudfront.net domain. It takes a few minutes to deploy. (It returns AccessDenied until you lock the bucket to CloudFront with OAC in L8.)
Invalidate the cache after a content change
aws cloudfront create-invalidation --distribution-id $DIST \
  --paths "/*"
Expect: an invalidation ID. First 1000 paths/month are free.
Fix / Key infoAdding an alternate CNAME + HTTPS needs a registered Route 53 domain + an ACM cert in us-east-1 (L4). Keep the $DIST + $BUCKET for L8.
Clean up (after L8)
# A distribution must be DISABLED then deleted (needs its ETag).
# Easiest in the console: Disable → wait → Delete. Then:
aws s3 rb s3://$BUCKET --force
Expect: bucket emptied + removed; disable/delete the distribution to stop it.

Stop people bypassing CloudFront.

Quick Understanding

MethodForHow it works
OAIS3 originsVirtual identity; bucket allows OAI only.

Explanation

  • Goal: make sure people can't bypass CloudFront and hit the origin directly.
  • S3 origins: use an OAI (legacy) — the bucket policy allows the OAI and denies everyone else (implicit deny).

  • OAC (L8) is the modern replacement for OAI.

Private Distributions & OAC

Signed access + the modern origin lock.

Quick Understanding

ConceptMeaningExam relevance
Signed URLTime-limited link to one file.Private single object.
Signed cookieAccess to many files.Whole area private.
Trusted key groupsModern signers (vs old trusted signers).No root keys.
OACNew OAI replacement; locks S3 to CloudFront.Recommended.

Key Terms

TermSimple meaning
Signed URL / cookieA signed token granting temporary access.
Trusted key groupThe public keys CloudFront trusts to verify signatures.
OACOrigin Access Control — newer, supports SSE-KMS + all regions.

Explanation

graph TD
  APP[App signs with private key] --> URL[Signed URL / cookie]
  URL --> CF[CloudFront verifies via Trusted Key Group]
  CF -->|valid| OBJ[Private object]
  CF -->|via OAC| S3[S3 locked to this distribution]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class CF a;
  • Make content private with signed URLs (one file) or signed cookies (many files); your app signs them with a private key and CloudFront verifies via a trusted key group.
  • Trusted key groups (new) replace trusted signers (old, needed account-level CloudFront keys + root).
  • OAC is the modern OAI replacement to lock an S3 origin to one distribution; it supports SSE-KMS and all regions.

Demo

Demo · CLI v2 (lock the L6 bucket with OAC)Situation: allow only your CloudFront distribution to read the S3 bucket. Check first: the $DIST + $BUCKET from L6. (Console is the easiest path; here are the key pieces.)
Create an Origin Access Control
OAC=$(aws cloudfront create-origin-access-control \
  --origin-access-control-config \
  'Name=a4l-oac,SigningProtocol=sigv4,SigningBehavior=always,OriginAccessControlOriginType=s3' \
  --query "OriginAccessControl.Id" --output text)
echo $OAC
Expect: an OAC ID. Attach it to the distribution's S3 origin (update the distribution config — easiest in the console).
Bucket policy: allow only this distribution
ACCT=$(aws sts get-caller-identity --query Account --output text)
cat > pol.json << EOF
{"Version":"2012-10-17","Statement":[{
 "Sid":"AllowCloudFrontOAC","Effect":"Allow",
 "Principal":{"Service":"cloudfront.amazonaws.com"},
 "Action":"s3:GetObject","Resource":"arn:aws:s3:::$BUCKET/*",
 "Condition":{"StringEquals":{"AWS:SourceArn":"arn:aws:cloudfront::$ACCT:distribution/$DIST"}}
}]}
EOF
aws s3api put-bucket-policy --bucket $BUCKET --policy file://pol.json
Expect: the bucket now serves only via that distribution; direct S3 access is denied.
Fix / Key infoPrefer OAC over OAI (OAC supports SSE-KMS + all regions). The SourceArn condition ties the bucket to one distribution.
Clean upRemove the bucket policy + OAC, then disable/delete the distribution (see L6 cleanup).
aws s3api delete-bucket-policy --bucket $BUCKET
aws cloudfront delete-origin-access-control --id $OAC --if-match \
  $(aws cloudfront get-origin-access-control --id $OAC --query ETag --output text)
rm -f pol.json index.html
Expect: policy + OAC removed.

How to useDecide first, then tap Show answer. Pick the best option. 18 questions.
1. Global users find a static site slow from one region. Best fix?
A) Bigger serverB) CloudFrontC) More RDS replicasD) NLB
B. CloudFront caches content near users to cut latency.
2. Your CloudFront cert (created in eu-west-1) is rejected. Why?
A) Must be in us-east-1B) Private certC) Wrong bucketD) Expired
A. CloudFront only uses ACM certs from us-east-1.
3. Update CSS often, want instant + free cache refresh. Best?
A) Invalidate /* each timeB) Versioned filenamesC) Disable cacheD) Lower TTL to 0
B. A new filename is a new cache key — instant, no invalidation cost.
4. Lock an S3 origin (SSE-KMS) so only one distribution can read it. Use?
A) OAIB) OACC) Public bucketD) Signed cookie
B. OAC supports SSE-KMS and is the modern OAI replacement.
5. Give temporary access to ONE private file via CloudFront. Use?
A) Signed URLB) Signed cookieC) OACD) Public
A. Signed URL = one file; signed cookie = many files.
6. A UDP game needs static IPs + fastest global path + fast failover. Use?
A) CloudFrontB) Global AcceleratorC) Route 53D) ALB
B. Global Accelerator gives static anycast IPs + optimised TCP/UDP paths.
7. Run auth + rewrite requests at the edge before the origin. Use?
A) Lambda@EdgeB) Step FunctionsC) SQSD) Glue
A. Lambda@Edge can modify the origin request at edge locations.
8. Force all CloudFront viewers onto HTTPS. Setting?
A) Behaviour viewer protocol policy = redirect-to-HTTPSB) New distributionC) OACD) Lower TTL
A. Set the behaviour to redirect HTTP to HTTPS.
9. (S3) Cheapest origin for a static website fronted by CloudFront?
A) EC2B) S3 bucketC) EFSD) RDS
B. S3 is the cheap, scalable static origin.
10. (Route 53) Point apex shop.a4l.com at a CloudFront distribution. Record?
A) CNAME at apexB) Alias A recordC) MXD) TXT
B. Alias works at the apex and points to AWS resources (free for CF targets).
11. (HA) CloudFront fronts an ALB + ASG web tier. Origin type for the ALB?
A) S3 REST originB) Custom originC) OAID) Gateway LB
B. An ALB is an HTTP server → custom origin.
12. (Serverless) CloudFront fronts an S3 SPA + API Gateway. How to split paths?
A) Two behaviours (S3 + API GW origins)B) Two distributionsC) Two bucketsD) Two certs
A. Path-pattern behaviours route /api/* to API GW and /* to S3.
13. (VPC) Stop users hitting a custom EC2 origin directly, bypassing CloudFront. Use?
A) Secret custom header + IP allow-listB) OAIC) Public SGD) Bigger instance
A. Custom origins are locked with a secret header + CloudFront IP allow-list (OAI is S3-only).
14. (IAM) ACM public certificate cost + renewal?
A) Free + auto-renewB) Paid + manualC) Free + manualD) Paid + auto
A. ACM public certs are free and auto-renewing.
15. (RDS) Can you install an ACM cert directly on an EC2 web server?
A) YesB) No — only CloudFront/ALB/API GWC) Only in us-east-1D) Only private certs
B. ACM certs deploy only to integrated services, not directly to EC2.
16. Many HTTPS domains on CloudFront with no extra cost. Use?
A) SNIB) Dedicated IP SSLC) No SSLD) Self-signed
A. SNI shares an IP across SSL sites for free on modern clients.
17. Default CloudFront TTL when the origin sends no cache headers?
A) 1 hourB) 24 hoursC) 0D) 7 days
B. Default TTL is 24 hours.
18. Simple, ultra-cheap viewer-side URL rewrite at the edge. Best?
A) CloudFront FunctionsB) Lambda@Edge origin responseC) Step FunctionsD) Glue
A. CloudFront Functions handle lightweight viewer-side changes very cheaply.
