Find meaning + insights inside text.

Quick Understanding

ConceptMeaningExam relevance

DetectsLanguage, sentiment, entities, key phrases, PII.What it outputs.
Input → insightGive text, get structured findings.Reviews, social.

Key Terms

TermSimple meaning
NLPNatural Language Processing — understanding text.
SentimentIs the text positive, negative, neutral?
EntityA named thing (person, place, date…).

Explanation

graph LR
  TXT[Text] --> COMP[Comprehend]
  COMP --> S[Sentiment]
  COMP --> E[Entities + key phrases]
  COMP --> L[Language + PII]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class COMP a;
  • Comprehend is an NLP service: give it text, get insights — sentiment, entities, key phrases, dominant language, PII, topics.
  • Use for analysing reviews, support tickets, social media, or finding personal data in text.

Demo

Demo · CLI v2 (tiny text, free tier)Situation: analyse a customer comment. Check first: CLI configured.
Detect sentiment
aws comprehend detect-sentiment --language-code en \
  --text "I absolutely love the Animals4life service, it is wonderful!"
Expect: Sentiment: POSITIVE + confidence scores.
Detect the language
aws comprehend detect-dominant-language \
  --text "Bonjour, je voudrais adopter un chat."
Expect: language code fr with a score.
Fix / Key infoNo infrastructure — you just call the API. Nothing to clean up (no resources created).

Build Knowledge

Connects to NOW
  • Text service alongside Translate (L8); Kendra (L2) for search.
Needed LATER
  • Text analytics pipelines.

Exam Pointers

Exam Tip
  • "Sentiment / entities / language from text" → Comprehend.
  • Don't confuse with Translate (language→language) or Kendra (search).

Amazon Polly

Turn text into lifelike speech.

Quick Understanding

ConceptMeaningExam relevance
Text → speech (TTS)Reads text aloud.Output is audio.
Lifelike voicesMany languages + voices.Natural sound.
SSMLControl pronunciation/pace.Fine tuning.

Key Terms

TermSimple meaning
TTSText-To-Speech.
SSMLMarkup to control how speech sounds.

Explanation

  • Polly converts text into lifelike speech (TTS); output is an audio file (mp3/ogg/pcm).
  • It supports many languages + voices and SSML for control.
  • Remember: Polly = text → speech; Transcribe (L7) = speech → text (opposites).

Demo

Demo · CLI v2 (tiny, free tier)Situation: make an audio greeting. Check first: CLI configured.
Synthesize speech to an mp3
aws polly synthesize-speech --output-format mp3 \
  --voice-id Joanna \
  --text "Hello from Animals for Life" hello.mp3
Expect: a hello.mp3 file you can play — Joanna reads the text aloud.
Fix / Key infoNo infrastructure created — just an output file. Polly is the opposite of Transcribe.
Clean up
rm -f hello.mp3
Expect: local file removed (nothing billable in AWS).

Build Knowledge

Connects to NOW
  • Voice output for Lex bots (L3); opposite of Transcribe (L7).
Needed LATER
  • Talking apps, accessibility.

Exam Pointers

Exam Tip
  • Polly = text → speech. Transcribe = speech → text. Don't swap them.

Flashcards

Explanation

  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;

  • SageMaker is a fully managed platform to build, train and deploy your own ML models.
  • Unlike the pre-trained services above, you bring the data + model; SageMaker handles notebooks, training, and hosted endpoints.
  • Key distinction: pre-built AI services (Comprehend, Rekognition…) solve known problems out of the box; SageMaker is for custom ML.
