AWS Secrets Manager

Store secrets and rotate them automatically.

Quick Understanding

ConceptMeaningExam relevance
Stores secretsDB creds, API keys; encrypted with KMS.Central + secure.
Auto-rotationLambda rotates creds on a schedule.Key feature.
DB integrationBuilt-in for RDS/Aurora/Redshift/DocumentDB.Changes the real cred.
vs Parameter StoreParameter Store is free; SM adds rotation.Cost trade-off.

Key Terms

TermSimple meaning
SecretSensitive data like a password or key.
RotationChanging a credential regularly for safety.
SSM Parameter StoreA simpler store for config + secrets (free tier).

Explanation

graph TD
  APP[App] -->|get secret via API| SM[Secrets Manager - KMS encrypted]
  SM -->|rotation| L[Lambda rotates the credential]
  L --> DB[(RDS/Aurora/etc - cred updated)]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class SM a;
  • Secrets Manager stores secrets (DB creds, API keys), encrypted with KMS, and apps fetch them via the API/SDK (with IAM control).
  • It can automatically rotate credentials using Lambda, and for RDS/Aurora/Redshift/DocumentDB it updates the actual service credential.
  • vs SSM Parameter Store: Parameter Store is free (good for config + basic secrets, no built-in rotation); Secrets Manager costs per secret but adds rotation + native DB integration.

Demo

Demo · CLI v2 (small monthly cost per secret)Situation: store + retrieve a DB credential. Check first: CLI configured.
Cost noteEach stored secret costs ~$0.40/month + API calls. We force-delete it at the end (no recovery window).
Create a secret
aws secretsmanager create-secret --name a4l/demo \
  --secret-string '{"username":"admin","password":"Sup3rS3cret!"}' \
  --query ARN --output text
Expect: a secret ARN. The value is stored encrypted (KMS).
Retrieve it (as an app would)
aws secretsmanager get-secret-value --secret-id a4l/demo \
  --query SecretString --output text
Expect: the JSON with username + password.
Fix / Key infoFor rotation, attach a Lambda (built-in templates exist for RDS). Use Parameter Store if you don't need rotation + want it free.
Clean up
aws secretsmanager delete-secret --secret-id a4l/demo \
  --force-delete-without-recovery
Expect: secret deleted immediately (force skips the 7–30 day recovery window).

Connects to NOW

Needed LATER
  • Secure app credential management.

Exam Pointers

Exam Tip
  • Need automatic credential rotation (esp. RDS) → Secrets Manager.
  • Just storing config / simple secrets cheaply → SSM Parameter Store.
  • Secrets are KMS-encrypted and fetched via IAM-controlled API.

Layer 7 (L7) Firewall

Seeing inside the application traffic.

Quick Understanding

ConceptMeaningExam relevance
OSI layersL3 = IP, L4 = port, L7 = application.What each sees.
L3/4 firewallSG/NACL: only IP + port.Blind to content.
L7 firewallSees HTTP content, headers, hostnames.Smart filtering.

Key Terms

TermSimple meaning
OSI modelA layered model of how networks work (L1–L7).
Layer 7The application layer (e.g. HTTP).
Payload / contentThe actual data inside a request.

Explanation

graph TD
  L34[L3/4 firewall - SG/NACL] --> SEE34[Sees only IP + port]
  L7[L7 firewall - e.g. WAF] --> SEE7[Sees content, headers, hostnames]
  SEE7 --> SMART[Block SQLi/bad bots, allow good traffic on same port]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class L7 a;
  • A L3/4 firewall (security group, NACL) sees only IP + port — it can't tell a good HTTPS request from an attack on the same port (443).
  • A L7 firewall sees inside the application data: HTTP content, headers, hostnames (for email, the metadata/contents).
  • This lets it block bad requests (SQL injection, bad bots) while allowing good traffic on the same port — the basis for WAF (L3).

Demo

No demo — theory onlyA concept about the OSI model; the hands-on L7 protection is WAF (L3).

Build Knowledge

Connects to NOW
  • Explains why WAF (L3) is needed beyond SG/NACL.
Needed LATER
  • Application protection design.

Exam Pointers

Exam Tip
  • SG/NACL = L3/4 (IP/port only). To filter on HTTP content you need a L7 firewall (WAF).

Flashcards

What can a L3/4 firewall see?tap
Only IP addresses and ports.
What extra can a L7 firewall see?tap
Inside the application data: HTTP content, headers, hostnames.

Mini Quiz

AWS's Layer 7 web firewall.

Quick Understanding

ConceptMeaningExam relevance
Web ACLThe main container of rules; attach to a resource.CloudFront/ALB/API GW.
Rule GroupReusable set of rules (AWS-managed or custom).Building block.
RuleCondition + action.SQLi, XSS, rate, geo, IP.
ActionsAllow / Block / Count / CAPTCHA.What to do.

Key Terms

TermSimple meaning
WAFWeb Application Firewall (Layer 7).
Web ACLThe set of rules WAF applies to a resource.
Rate-based ruleBlocks an IP that sends too many requests.

Explanation

graph TD
  WEBACL[Web ACL] --> RG[Rule Groups - managed/custom]
  RG --> RULE[Rules: SQLi / XSS / rate / geo / IP]
  WEBACL --> ATTACH[Attach to CloudFront / ALB / API Gateway]
  RULE --> ACT[Action: Allow / Block / Count / CAPTCHA]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class WEBACL a;
  • WAF protects web apps/APIs from common exploits (SQLi, XSS) and bad bots at Layer 7.
  • A Web ACL is the container you attach to CloudFront, ALB, API Gateway, AppSync; it holds Rule Groups (AWS-managed or custom) made of Rules.
  • Rules can match SQLi/XSS, rate limits, geo, IP sets; actions are Allow/Block/Count/CAPTCHA.

Quick Understanding

TierWhatExam relevance

Shield AdvancedPaid; adds L7, WAF, DRT, cost protection.~$3000/month.

Key Terms

TermSimple meaning
DDoSFlooding a service with traffic to take it down.
DRT/SRTAWS's DDoS Response Team (Advanced only).
Cost protectionRefund of scaling charges caused by an attack.

Explanation

  • Shield Standard is free and automatic for all AWS customers, protecting against common network/transport (L3/4) DDoS attacks (especially via CloudFront + Route 53).

  • Advanced protects CloudFront, Route 53, ELB, Global Accelerator, Elastic IPs.

CloudHSM

Your own dedicated key hardware.

Quick Understanding

ConceptMeaningExam relevance
Single-tenant HSMDedicated hardware just for you.Full control.
FIPS 140-2 Level 3A strict security standard.Compliance driver.
You own the keysAWS can't access them.vs KMS (shared).
Industry APIsPKCS#11, JCE, CNG (not native AWS).Access method.

Key Terms

TermSimple meaning
HSMHardware Security Module — a secure box for keys.
Single-tenantHardware used only by you (not shared).
FIPS 140-2 L3A high-bar cryptographic security standard.

Explanation

graph TD
  KMS[KMS - shared, AWS-managed, native API] --> EASY[Easy, integrated]
  HSM[CloudHSM - single-tenant, you own keys] --> COMP[FIPS 140-2 L3 + full control]
  HSM --> IND[Access via PKCS#11 / JCE / CNG]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class HSM a;
  • CloudHSM is a dedicated, single-tenant Hardware Security Module — only you have access; AWS manages the hardware but can't access your keys.
  • It meets FIPS 140-2 Level 3 and is accessed via industry-standard APIs (PKCS#11, JCE, CNG), not native AWS APIs.
  • vs KMS: KMS is shared/AWS-managed and easy/native; CloudHSM gives full control for strict compliance. KMS can use CloudHSM as a custom key store.

Key Terms

TermSimple meaning

Config ruleA check for whether a resource is compliant.

Explanation

graph TD
  RES[Resources] --> CFG[Config records configuration items]
  CFG --> HIST[Config history in S3]
  CFG --> RULE[Config rules check compliance]
  RULE --> NOTIFY[EventBridge / SNS on non-compliance]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class CFG a;
  • AWS Config records the configuration of resources over time as configuration items, kept as a history in an S3 bucket (regional).
  • It checks compliance with rules (AWS-managed or custom Lambda) and can send notifications/events (EventBridge/SNS).
  • Important: Config does not prevent or block changes — it only records and flags. [added for exam]

Key Terms

TermSimple meaning
PIIPersonal data (names, card numbers, etc.).
Managed data identifierBuilt-in detector for common sensitive data.

Explanation

  • Macie uses ML + pattern matching to discover and protect sensitive data (PII) in S3.

Key Terms

Threat detectionFinding signs of an attack/compromise.
FindingAn alert about suspicious activity.

Explanation

graph TD
  SRC[VPC Flow Logs + DNS logs + CloudTrail] --> GD[GuardDuty - ML baseline]
  GD --> FIND[Findings: crypto-mining, recon, odd API calls]
  FIND --> EB[EventBridge → Lambda auto-remediation]
  classDef a fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class GD a;
  • GuardDuty continuously analyses logs (VPC Flow Logs, DNS logs, CloudTrail events; also S3/EKS) and uses ML to learn what's normal.
  • It flags threats/anomalies (crypto-mining, compromised instances, recon, unusual API calls) as findings.
  • Findings can trigger EventBridge → Lambda for automatic remediation; works across multiple accounts.
