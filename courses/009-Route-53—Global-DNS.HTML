Public Hosted Zones

DNS records the whole internet can resolve.

Quick Understanding

ConceptMeaningExam relevance
Hosted zoneA container of DNS records for one domain.~$0.50/month each.
PublicResolvable from the internet (and VPCs).Internet-facing.
Name serversR53 gives 4 NS records for the zone.Delegation point.

Key Terms

TermSimple meaning
DNSThe system that turns names into IP addresses.
RecordOne DNS entry (A, AAAA, CNAME, MX, NS…).
NS recordPoints a domain to the name servers that answer for it.
DelegationThe parent zone hands control to your name servers.

Explanation

graph TD
  USER[User asks for www.a4l.com] --> RES[DNS resolver]
  RES --> TLD[.com tells it the NS for the zone]
  TLD --> R53[Route 53 public hosted zone]
  R53 --> REC[Returns the A record value]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class R53 a;
  • A public hosted zone holds the records for a domain and is resolvable from the internet.
  • R53 gives the zone 4 name servers (NS). Your registrar points the domain at them (delegation).
  • Each hosted zone costs about $0.50/month + small per-query charges.

Explanation

  • The VPC needs DNS support + DNS hostnames enabled.

Demo

Small costHosted zone ~$0.50/month — delete after.
Demo · CLI v2Situation: create a private zone tied to a VPC. Check first: a VPC ID (default VPC fine) in $VPC.
Get a VPC, then create the private zone
VPC=$(aws ec2 describe-vpcs --filters Name=isDefault,Values=true \
  --query "Vpcs[0].VpcId" --output text)
REGION=us-east-1
PZID=$(aws route53 create-hosted-zone --name a4l.internal \
  --caller-reference $(date +%s) \
  --vpc VPCRegion=$REGION,VPCId=$VPC \
  --hosted-zone-config PrivateZone=true \
  --query "HostedZone.Id" --output text)
echo $PZID
Expect: a private zone ID. Records here resolve only from inside $VPC.
Fix / Key infoAdd records the same way as a public zone. From an instance in the VPC, nslookup something.a4l.internal resolves; from the internet it doesn't.
Clean up
aws route53 delete-hosted-zone --id $PZID
Expect: zone deleted (remove any custom records first).

CNAME vs R53 Alias

Why Alias exists — and when to use it.

Quick Understanding

ConceptMeaningExam relevance
CNAMEMaps a name to another name.Not at apex; costs a query.
AliasR53 record pointing to an AWS resource.Works at apex; free.
Apex / naked domainThe root, e.g. a4l.com (no www).CNAME can't go here.

Key Terms

TermSimple meaning
ApexThe bare domain with no subdomain part.
Alias targetAn AWS resource: ELB, CloudFront, S3 site, API GW, etc.

Explanation

graph TD
  Q{Pointing at an AWS resource or the apex?} -->|Yes| ALIAS[Use ALIAS - works at apex, free queries to AWS]
  Q -->|No, just another DNS name on a subdomain| CNAME[CNAME is fine]
  CNAME --> X[CANNOT be used at the zone apex]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class ALIAS a;
  • CNAME maps a name to another name. It cannot exist at the zone apex (naked domain) and you pay for the query.
  • Alias is an R53 extension that maps a name (including the apex) to an AWS resource (ELB, CloudFront, S3 website, API Gateway, another R53 record…). Queries to AWS targets are free.
  • Rule of thumb: pointing at an AWS resource → use Alias (especially at the apex).

Fix / Key infoAn Alias record uses AliasTarget (the AWS resource's hosted-zone-id + DNS name) instead of ResourceRecords, and it's allowed at the apex. You need a real AWS target (e.g. an ELB) to create one.
Clean up
sed 's/UPSERT/DELETE/' cname.json > cnamedel.json
aws route53 change-resource-record-sets --hosted-zone-id $ZONEID --change-batch file://cnamedel.json

  • Alias points to AWS resources and is free for those queries.
  • "Point a naked domain at an ELB/CloudFront/S3" → Alias.

The default — one record, all values returned.

Quick Understanding

ConceptMeaningExam relevance
Simple routingOne record name; can hold multiple values.Default policy.
All values returnedClient picks one at random.No smart routing.
No health checksCan't detect a dead resource.Key limitation.

Key Terms

TermSimple meaning
TTLHow long resolvers cache the answer (seconds).
Record valueOne IP/target stored in the record.

Explanation

  • Simple routing uses one record that can have several values; R53 returns them all and the client picks one.
  • There are no health checks — if a value is dead, clients can still get it.
  • Good for a single resource; use other policies when you need health awareness or smart steering.

Route 53 Health Checks

Monitor resources so routing can react.

Quick Understanding

ConceptMeaningExam relevance
What it checksAn endpoint, other health checks, or a CW alarm.Three types.
Global checkersChecked from many AWS locations.Not from your VPC.
Drives routingPolicies skip unhealthy records.Health-aware DNS.

Key Terms

TermSimple meaning
Endpoint checkR53 connects to an IP/domain (TCP/HTTP/HTTPS).
Calculated checkA check whose health depends on other checks.
CloudWatch alarm checkHealth follows a CW alarm state (for private resources).

Explanation

graph TD
  HC{Health check type} --> EP[Endpoint: TCP/HTTP/HTTPS + optional string]
  HC --> CALC[Calculated: combine other checks]
  HC --> CWA[CloudWatch alarm: e.g. for private resources]
  EP --> ROUTE[Routing policies use health to include/exclude records]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class HC a;
  • Health checks run from global AWS checkers (not your VPC) every 30s (or 10s fast, extra cost).
  • Types: endpoint (TCP/HTTP/HTTPS, can match a string), calculated (based on other checks), CloudWatch alarm (good for private resources the global checkers can't reach).
  • A resource is healthy when enough checkers agree; routing policies then include only healthy records.

Demo

Small costA basic health check is ~$0.50/month. Delete after the failover/multi-value demos that use it.
Demo · CLI v2Situation: create an HTTP health check and read its status. Check first: CLI configured.
Create an HTTP health check
HC=$(aws route53 create-health-check \
  --caller-reference hc-$(date +%s) \
  --health-check-config Type=HTTP,FullyQualifiedDomainName=example.com,Port=80,RequestInterval=30,FailureThreshold=3 \
  --query "HealthCheck.Id" --output text)
echo $HC
Expect: a health-check ID like abcd-1234.
Check its status (give it a minute)
aws route53 get-health-check-status --health-check-id $HC \
  --query "HealthCheckObservations[].StatusReport.Status"
Expect: status lines from several global locations (Success/Failure).
Fix / Key infoFor a private resource (no public endpoint), point a health check at a CloudWatch alarm instead. Keep $HC for L6/L7.
Clean up (after L6/L7)
aws route53 delete-health-check --health-check-id $HC
Expect: health check deleted (detach it from any records first).

Build Knowledge

Connects to NOW
  • Used by failover (L6) and multi-value (L7).
Needed LATER
  • Highly available, self-healing DNS designs.

Exam Pointers

Exam Tip
  • Three types: endpoint, calculated, CloudWatch alarm.
  • Checkers are global (not in your VPC) → for private resources use the CW alarm type.
  • Intervals: 30s standard, 10s fast (extra cost).

Flashcards

Key Terms

TermSimple meaning

Explanation

WeightA number setting how much traffic a record gets.

No match + no defaultR53 returns "no answer".Common trap.

Key Terms

TermSimple meaning
GeoLocationCountry / continent / default tag on a record.
Default recordUsed when no specific location matches.

Explanation

graph TD
  U[User location] --> M{Most specific match?}
  M -->|Country| C[Country record]
  M -->|else Continent| K[Continent record]
  M -->|else| D[Default record - or NO ANSWER]

  • Used for localisation (language/site per country) and compliance/geo-restriction. It's about the user's location, not the nearest region.

DNSSEC with Route 53

Sign DNS data so it can't be forged.

Quick Understanding

ConceptMeaningExam relevance
DNSSECDigitally signs DNS data to stop spoofing.Authenticity, not secrecy.
KSK + KMSKey Signing Key backed by a KMS asymmetric key.Uses KMS (us-east-1).
Chain of trustParent zone holds a DS record pointing to your zone.Must add DS at parent.

Key Terms

TermSimple meaning
DNSSECSignatures that prove a DNS answer is genuine.
KSK / ZSKKey Signing Key / Zone Signing Key used to sign records.
DS recordThe fingerprint placed in the parent zone to build trust.

Explanation

graph TD
  KMS[KMS asymmetric key us-east-1] --> KSK[Key Signing Key]
  KSK --> SIGN[Enable DNSSEC signing on the zone]
  SIGN --> ZONE[Records are signed]
  ZONE --> DS[Add DS record to PARENT zone = chain of trust]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class KMS a;
  • DNSSEC signs DNS data so resolvers can verify it's genuine (stops spoofing/cache poisoning). It does not encrypt queries.
  • In R53 you create a KSK backed by a KMS asymmetric key in us-east-1, then enable signing on the hosted zone.
  • Finally you add a DS record at the parent zone (via the registrar) to complete the chain of trust.

Demo

Cost + regionNeeds a KMS asymmetric key in us-east-1 (~$1/month). Schedule the key for deletion in cleanup. Public resolution needs a real delegated domain.
Demo · CLI v2Situation: enable DNSSEC signing on a hosted zone. Check first: a public zone ID in $ZONEID (from L1).
Create the KMS key for the KSK (us-east-1, asymmetric)
KEY=$(aws kms create-key --region us-east-1 \
  --key-spec ECC_NIST_P256 --key-usage SIGN_VERIFY \
  --query "KeyMetadata.Arn" --output text)
echo $KEY
Expect: a KMS key ARN. (R53 DNSSEC requires the key in us-east-1.)
Create a Key Signing Key + enable signing
aws route53 create-key-signing-key --caller-reference ksk-$(date +%s) \
  --hosted-zone-id $ZONEID --key-management-service-arn $KEY \
  --name a4lksk --status ACTIVE
aws route53 enable-hosted-zone-dnssec --hosted-zone-id $ZONEID
Expect: a KSK created and signing enabled. R53 now signs the zone's records.
Get the DS record to give your parent/registrar
aws route53 get-dnssec --hosted-zone-id $ZONEID \
  --query "KeySigningKeys[0].DSRecord"
Expect: a DS record string. You add this at the parent (registrar) to finish the chain of trust.
Fix / Key infoDNSSEC = authenticity, not encryption. The KMS key must be in us-east-1. Trust is only complete once the DS record is at the parent.
Clean up
aws route53 disable-hosted-zone-dnssec --hosted-zone-id $ZONEID
aws route53 deactivate-key-signing-key --hosted-zone-id $ZONEID --name a4lksk
aws route53 delete-key-signing-key --hosted-zone-id $ZONEID --name a4lksk
aws kms schedule-key-deletion --region us-east-1 --key-id $KEY --pending-window-in-days 7
Expect: signing disabled, KSK removed, KMS key scheduled for deletion (7-day window — no instant delete).
