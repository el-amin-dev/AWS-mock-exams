Virtualization 101

How many virtual machines share one physical server.

Quick Understanding

ConceptMeaningExam relevance
VirtualizationRunning many virtual machines on one physical host.Background for EC2.
Hardware-assistedCPU helps the hypervisor → faster, modern.What AWS uses today.
SR-IOVThe network card presents virtual NICs to instances.= Enhanced Networking.

Key Terms

TermSimple meaning
HypervisorSoftware that creates and runs virtual machines.
EmulationSoftware pretends to be hardware (slow).
ParavirtualizationThe guest OS is modified to cooperate with the hypervisor.
SR-IOVHardware lets one physical card act as many virtual cards.
NitroAWS's modern system that offloads work to dedicated hardware.

Explanation

graph TD
  EMU[Emulated: software fakes hardware - slow] --> PARA[Paravirtual: guest cooperates - faster]
  PARA --> HW[Hardware-assisted: CPU helps - fast]
  HW --> SRIOV[SR-IOV: network card offload = Enhanced Networking]
  SRIOV --> NITRO[AWS Nitro: offloads compute/storage/network to hardware]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class NITRO a;
  • Old way (emulation) was slow; paravirtualization sped it up; hardware-assisted is the modern base.
  • SR-IOV lets the physical network card present virtual NICs to instances → this is Enhanced Networking (lower latency, higher throughput, less host overhead).
  • AWS now uses the Nitro system, offloading networking/storage to dedicated hardware. [added for exam]

EC2 Architecture & Resilience

Where instances run and why the AZ matters.

Quick Understanding

ConceptMeaningExam relevance
EC2 HostA physical server in ONE AZ that runs instances.Instances are AZ-locked.
AZ resilienceAn instance fails if its AZ fails.Spread across AZs for HA.
EBS in same AZAn instance and its EBS volume share one AZ.Can't attach across AZs.
Stop/Start = moveStopping then starting can move to a new host.Fixes host issues.

Key Terms

TermSimple meaning
InstanceA virtual server running on a host.
EC2 HostThe physical machine the instance runs on.
AZAvailability Zone — an isolated data-center area in a region.
EBSNetwork block storage that lives in one AZ.

Explanation

graph TD
  REGION[Region] --> AZA[AZ-a]
  REGION --> AZB[AZ-b]
  AZA --> HOST[EC2 Host]
  HOST --> INST[Instance]
  INST --- EBS[EBS volume - same AZ]
  AZA -. if AZ-a fails .- DOWN[Instance + its EBS go down]
  classDef w fill:#3a151c,stroke:#fb7185,color:#e7edf3;
  class DOWN w;
  • Instances run on an EC2 host inside one AZ — they don't move between AZs by themselves.
  • An instance and its EBS volumes must be in the same AZ.
  • If the AZ fails, the instance fails → for HA, run instances across multiple AZs.
  • Stop then start can move an instance to a different host (useful to escape a failing host); a reboot keeps the same host.

Storage Refresher

The words behind every storage choice.

Quick Understanding

ConceptMeaningExam relevance
Block storageRaw disk you format + mount (EBS, instance store).EC2 boot/data volumes.
File storageA shared network file system (EFS, FSx).Many clients share files.
Object storageFlat store of objects via API (S3).Not mountable like a disk.
IOPS vs throughputOperations/sec vs amount of data/sec.Pick the right metric.

Key Terms

TermSimple meaning
IO block sizeThe size of each read/write chunk.
IOPSInput/Output operations per second (how many small ops).
ThroughputHow much data moves per second (MB/s).

Explanation

graph TD
  BLOCK[Block - EBS / Instance Store] --> MOUNT[Format + mount as a drive]
  FILE[File - EFS / FSx] --> SHARE[Many clients share over network]
  OBJ[Object - S3] --> API[Access by API, not mounted]
  IOPS[IOPS x block size ≈ throughput]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class BLOCK a;
  • Block = a raw disk you format and mount (EBS, instance store).
  • File = a shared file system many machines mount (EFS, FSx).
  • Object = store/retrieve whole objects by API (S3) — not a mounted disk.
  • IOPS matters for many small operations (databases); throughput matters for big sequential data. Roughly: IOPS × block size ≈ throughput.

EBS Service Architecture

Network block storage that outlives the instance.

Quick Understanding

ConceptMeaningExam relevance
EBS volumeA raw block device you attach to an instance.Main EC2 storage.
AZ-scopedLives in one AZ; replicated within that AZ.Same AZ as instance.
PersistentSurvives instance stop/terminate (if not delete-on-terminate).Data stays.
Snapshot to S3Back up to S3; move across AZ/region.Migration path.

Key Terms

TermSimple meaning
VolumeA virtual disk you attach to an instance.
Attach / detachConnect or disconnect a volume from an instance.
Block deviceA disk the OS can format and use.
Multi-attachio1/io2 can attach to several instances at once (special case).

Explanation

graph TD
  AZ[AZ-a] --> EBS[EBS volume - replicated in AZ-a]
  EBS -->|attach| INST[Instance in AZ-a]
  EBS -->|snapshot| S3[(S3 - move across AZ/region)]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class EBS a;
  • EBS is network-attached block storage, living in one AZ (replicated inside that AZ).
  • Volumes are persistent — they keep data when an instance stops, and can be detached/re-attached.
  • Usually one volume attaches to one instance; io1/io2 multi-attach is the exception.
  • Back up via snapshots (to S3) to move data across AZs or regions.

Demo

Demo · CLI v2Situation: create a small EBS volume and inspect it. Check first: CLI configured; pick an AZ (us-east-1a).
Create a 8 GiB gp3 volume
VOL=$(aws ec2 create-volume --availability-zone us-east-1a \
  --size 8 --volume-type gp3 \
  --tag-specifications 'ResourceType=volume,Tags=[{Key=Name,Value=a4l-vol}]' \
  --query VolumeId --output text)
echo $VOL
Expect: a volume ID like vol-0abc....
See its state + AZ
aws ec2 describe-volumes --volume-ids $VOL \
  --query "Volumes[0].{State:State,AZ:AvailabilityZone,Type:VolumeType}"
Expect: state available, AZ us-east-1a, type gp3.
Fix / Key infoA volume can only attach to an instance in the same AZ. The full attach/mount/snapshot flow is in Lesson 11.

Expect: no output. (A small gp3 volume costs pennies/day; delete to be safe.)

EBS Volume Types — General Purpose

gp2 and gp3 SSD for most workloads.

Quick Understanding

ConceptMeaningExam relevance
gp2SSD; IOPS tied to size (3 IOPS/GB), burst credits.Older default.
gp3SSD; 3000 IOPS + 125 MB/s baseline, scale separately.Cheaper + flexible.
Default choiceGeneral purpose suits most workloads."Balanced" answer.

Key Terms

TermSimple meaning
BaselineThe performance you always get.
Burst creditsgp2 saves credits when idle to spend on spikes.
Decouplegp3 lets you set IOPS/throughput independent of size.

Explanation

  • gp2: IOPS scale with size (3 IOPS per GB), up to 16,000 IOPS; small volumes use a burst credit pool.
  • gp3: starts at 3,000 IOPS + 125 MB/s regardless of size; you can pay to raise to 16,000 IOPS / 1,000 MB/s independently.
  • gp3 is usually cheaper than gp2 for the same baseline — default to gp3.
  • Both are SSD, good for boot volumes and most general workloads.

Demo

Demo · CLI v2Situation: create a gp3 volume with extra IOPS/throughput. Check first: CLI configured.
Create gp3 with 4000 IOPS + 200 MB/s
VOL=$(aws ec2 create-volume --availability-zone us-east-1a \
  --size 20 --volume-type gp3 --iops 4000 --throughput 200 \
  --query VolumeId --output text)
aws ec2 describe-volumes --volume-ids $VOL \
  --query "Volumes[0].{Iops:Iops,Tput:Throughput,Type:VolumeType}"
Expect: IOPS 4000, throughput 200, type gp3 — performance set independent of the 20 GiB size.
Fix / Key infoWith gp2 you'd need a much bigger volume to reach 4000 IOPS; gp3 decouples size from performance.
Clean up
aws ec2 delete-volume --volume-id $VOL
Expect: no output.

io1 / io2 for the highest, steadiest performance.

Quick Understanding

ConceptMeaningExam relevance
Provisioned IOPSYou set an exact, high IOPS number.Mission-critical DBs.
High IOPS:GBLots of IOPS without a huge volume.io2 up to 1000:1.

Key Terms

TermSimple meaning
ProvisionedYou request and pay for a set performance level.
LatencyThe delay before each operation completes.
Block ExpressThe fastest io2 tier (very high IOPS + low latency).

Explanation

  • io1/io2 let you provision a precise, high IOPS level for consistent low latency.

  • Support multi-attach (share one volume across instances).

Key Terms

TermSimple meaning
SequentialReading/writing data in long continuous streams.

Explanation

Key Terms

TermSimple meaning

Explanation

graph TD
  HOST[EC2 Host] --> IS[Instance Store - local disk]
  IS --> REBOOT[Reboot: data SURVIVES - same host]

  classDef w fill:#3a151c,stroke:#fb7185,color:#e7edf3;

  • Instance store is local to the host → very high performance, included in the instance price.
  • It's ephemeral: data survives a reboot but is lost on stop/start, terminate, or host failure.
  • Must be attached at launch (can't add later); size/number depend on the instance type.
  • Great for caches, scratch, temporary data that you can lose or rebuild.

Explanation

  APP[App on instance] -->|GET token then data| MD[169.254.169.254/latest/meta-data]

  MD --> ROLE[iam/security-credentials = temp role creds]

  • Reach metadata at http://169.254.169.254/latest/meta-data/ from inside the instance.
  • It exposes hostname, IPs, AZ, security groups, and the IAM role's temporary credentials (this is how SDKs/CLI on the instance authenticate).
  • IMDSv2 requires a session token first → use it (more secure than IMDSv1).
  • Metadata has no authentication by default, so never put secrets in user-data.

  -H "X-aws-ec2-metadata-token-ttl-seconds: 300")
