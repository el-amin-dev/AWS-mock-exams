S3 Security (Resource Policies & ACLs)

Three ways to control access — and which to use.

Quick Understanding

ConceptMeaningExam relevance
Identity policyAttached to an IAM user/role; controls what they can do.Same-account access.
Bucket policyResource policy on the bucket; can allow anonymous + cross-account.Most flexible — favourite.
ACL (legacy)Old per-object/bucket access list. Avoid."Don't use ACLs" trap.
Block Public AccessMaster safety switch; blocks public even if policy allows.Overrides everything.

Key Terms

TermSimple meaning
Resource policyA policy attached to the resource (the bucket), naming who can use it.
PrincipalThe "who" — an account, user, role, or * (anyone).
AnonymousA request with no AWS identity (the public).
ACLAccess Control List — a simple, legacy grant list.

Explanation

graph TD
  Q{Who needs access?} --> A[Your IAM users, many buckets] --> IP[Use IDENTITY policy]
  Q --> B[Anonymous public OR another account] --> BP[Use BUCKET policy]
  Q --> C[Old per-object grants] --> ACL[ACL - legacy, avoid]
  BPA[Block Public Access] -. can override .-> BP
  classDef g fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  classDef w fill:#3a151c,stroke:#fb7185,color:#e7edf3;
  class IP,BP g; class ACL w;
  • Identity policy: best when controlling your own account's users across many resources.
  • Bucket policy: needed for anonymous access or cross-account, and to set rules for a whole bucket.
  • ACLs are legacy — AWS recommends not using them (now off by default).
  • Block Public Access (BPA) is a safety net: if on, public access is blocked even if a policy allows it.

Demo

Demo · CLI v2Situation: add a bucket policy that allows only your own account to read, then view it. Check first: CLI configured as admin.
Make a bucket + get your account ID
ACC=$(aws sts get-caller-identity --query Account --output text)

aws s3 mb s3://$BUCKET --region us-east-1
Expect: make_bucket success.
Apply a bucket policy (resource policy)
cat > bp.json <
Expect: no output = policy set. Read it back: aws s3api get-bucket-policy --bucket $BUCKET.
Fix / Key infoWe allowed our own account here (safe). Setting Principal:"*" would try public access — but Block Public Access would still block it unless turned off.
Clean up

rm -f bp.json

Connects to NOW

  • Encryption (L6) and access points (L16) layer on top of this.
Needed LATER
  • CloudFront OAC, VPC endpoint policies, cross-account data sharing.

Exam Pointers

Exam Tip
  • Anonymous or cross-account → bucket policy (identity policies can't grant to anonymous).
  • Block Public Access overrides any allow — check it first when "bucket is still private".
  • ACLs are legacy; prefer policies.

Explanation

  USER[Visitor] --> EP[S3 Website Endpoint - HTTP]
  EP --> IDX[index.html]
  EP --> ERR[error.html on failure]
  NOTE[Needs: BPA off + public bucket policy] -. setup .-> EP

  • Enable hosting, set index and error documents.
  • To be public: turn off Block Public Access + add a public-read bucket policy.

  • Great for cheap sites, offloading, and failover / maintenance pages.

Demo

Demo · CLI v2Situation: host the Animals4life "Top 10" page. Check first: CLI configured. Note: this makes a bucket public — clean up after.
Create bucket + turn off Block Public Access
BUCKET=a4l-top10-$RANDOM$RANDOM
aws s3 mb s3://$BUCKET --region us-east-1
aws s3api put-public-access-block --bucket $BUCKET \
  --public-access-block-configuration BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false
Expect: bucket made; BPA turned off (needed for a public website).
Add index + error pages and upload
echo "<h1>A4L Top 10 Animals</h1>" > index.html
echo "<h1>Oops - not found</h1>" > error.html
aws s3 cp index.html s3://$BUCKET/
aws s3 cp error.html s3://$BUCKET/
Expect: two uploads succeed.
Enable website + public-read policy
aws s3 website s3://$BUCKET/ --index-document index.html --error-document error.html
cat > pub.json <
Expect: a website URL printed — open it in a browser to see the page (HTTP only).
Fix / Key infoThe website endpoint (s3-website-...) differs from the normal REST endpoint. It serves index/error docs and is HTTP only.
Clean up
aws s3 rb s3://$BUCKET --force
rm -f index.html error.html pub.json
Expect: bucket + files removed. No cost.

Object Versioning & MFA Delete

Keep old versions; recover from mistakes.

Quick Understanding

ConceptMeaningExam relevance
VersioningBucket keeps every version of an object.Recovery + replication need it.
Delete markerDelete just hides the object with a marker; data stays.Common trap.
Can't disableOnce enabled, you can only Suspend, not turn off.Detail question.
MFA DeleteRequires MFA to change versioning / delete versions.Extra protection.

Key Terms

TermSimple meaning
Version IDA unique ID for each version of an object.
Current versionThe newest version (returned when no ID given).
Delete markerA special "latest version" that hides the object.
SuspendStop making new versions, but keep existing ones.

Explanation

graph TD
  P1[Upload v1] --> P2[Upload v2 - now current]
  P2 --> DEL[Delete object]
  DEL --> DM[Delete marker = current
v1 + v2 still stored]
  DM --> REC[Remove marker = object back]
  classDef k fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class DM k;
  • With versioning on, each upload makes a new version; the newest is the current one.
  • A plain delete just adds a delete marker — old versions stay (and still cost storage).
  • Restore by deleting the delete marker.
  • Versioning can be Enabled or Suspended, never fully off again.
  • MFA Delete forces MFA to suspend versioning or permanently delete versions.

Demo

Demo · CLI v2Situation: "Animal of the week" page — recover after an accidental delete. Check first: CLI configured.
Create bucket + enable versioning
BUCKET=a4l-ver-$RANDOM$RANDOM
aws s3 mb s3://$BUCKET --region us-east-1
aws s3api put-bucket-versioning --bucket $BUCKET \
  --versioning-configuration Status=Enabled
Expect: bucket made; versioning enabled (no output).
Upload two versions, then delete
echo "cat v1" > animal.txt && aws s3 cp animal.txt s3://$BUCKET/animal.txt
echo "cat v2" > animal.txt && aws s3 cp animal.txt s3://$BUCKET/animal.txt
aws s3 rm s3://$BUCKET/animal.txt
Expect: two uploads; the rm adds a delete marker (object now "gone").
List all versions (see the marker)
aws s3api list-object-versions --bucket $BUCKET \
  --query "{Versions:Versions[].VersionId,Markers:DeleteMarkers[].VersionId}"
Expect: two version IDs + one delete-marker ID — proof nothing was truly deleted.
Recover by deleting the delete marker
MID=$(aws s3api list-object-versions --bucket $BUCKET \
  --query "DeleteMarkers[0].VersionId" --output text)
aws s3api delete-object --bucket $BUCKET --key animal.txt --version-id $MID
aws s3 cp s3://$BUCKET/animal.txt -   # prints "cat v2"
Expect: the object is back; reading it shows cat v2.
Fix / Key infoOld versions cost storage until removed. Use lifecycle rules (L9) to expire non-current versions.
Clean uprb --force deletes all versions + markers:
aws s3 rb s3://$BUCKET --force
rm -f animal.txt
Expect: bucket and all versions removed.

Build Knowledge

Connects to NOW
  • Replication (L10) and Object Lock (L15) require versioning.
  • Lifecycle (L9) cleans up old versions.
Needed LATER
  • Backup/DR strategies built on versioned buckets.

Exam Pointers

Exam Tip
  • Delete with versioning = a delete marker; data isn't gone.
  • Versioning can be suspended but never disabled.
  • Old versions still cost money.
  • MFA Delete = MFA required to suspend versioning or delete versions.

Flashcards

What happens on delete with versioning on?tap
A delete marker is added; older versions remain stored.
Can you turn versioning off?tap
No — only suspend it. Existing versions stay.
How to recover a "deleted" object?tap
Delete its delete marker.
What does MFA Delete protect?tap
Suspending versioning and permanently deleting versions.

Mini Quiz

A) Re-upload itB) Delete the delete markerC) Disable versioning
B. The data still exists; removing the delete marker brings it back.

S3 Performance Optimization

Faster, more reliable uploads.

Quick Understanding

ConceptMeaningExam relevance
Single PUTUpload an object in one request (max 5 GB).Simple but fragile for big files.
Multipart uploadSplit into parts, upload in parallel, reassemble.Big files / reliability.
Transfer AccelerationUse edge + AWS backbone for fast long-distance uploads.Remote users, big data.

Key Terms

TermSimple meaning
MultipartBreaking a big upload into smaller pieces.
PartOne piece of a multipart upload.
Transfer Acceleration (TA)Speeds far-away uploads via the nearest edge location.
Edge locationAn AWS site close to users (also used by CloudFront).

Explanation

graph TD
  F[File to upload] --> SZ{Size?}
  SZ -->|Up to 5GB| SP[Single PUT]
  SZ -->|Large / want resilience| MP[Multipart: parallel parts]
  MP --> FAST[Faster + retry single parts]
  FAR[User far from bucket region] --> TA[Transfer Acceleration via edge]
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class MP,TA k;
  • Single PUT max 5 GB. For bigger or more reliable uploads use multipart.
  • Multipart: recommended over 100 MB, required over 5 GB; up to 10,000 parts, uploaded in parallel; a failed part can be retried alone.
  • Transfer Acceleration sends data to the nearest edge, then over AWS's fast network — great for remote users and large datasets (extra cost).
  • TA needs a DNS-compatible bucket name with no dots.

Demo

Demo · CLI v2Situation: enable Transfer Acceleration for Animals4life remote workers. Check first: CLI configured; bucket name has no dots.
Create a bucket + enable acceleration
BUCKET=a4l-accel-$RANDOM$RANDOM
aws s3 mb s3://$BUCKET --region us-east-1
aws s3api put-bucket-accelerate-configuration \
  --bucket $BUCKET --accelerate-configuration Status=Enabled
Expect: no output = acceleration on.
Confirm it's enabled
aws s3api get-bucket-accelerate-configuration --bucket $BUCKET
Expect: { "Status": "Enabled" }. The accelerate endpoint is $BUCKET.s3-accelerate.amazonaws.com.
Fix / Key infoThe AWS speed-comparison tool shows direct vs accelerated upload times from your location. TA helps most when you're far from the bucket's region.
Clean up
aws s3 rb s3://$BUCKET --force
Expect: bucket removed. (Enabling TA itself has no cost; only accelerated transfers cost extra.)

Build Knowledge

Connects to NOW
  • Edge locations also power CloudFront (later).
Needed LATER
  • CloudFront, global delivery, large data ingestion patterns.

Exam Pointers

Exam Tip
  • Single PUT limit = 5 GB; large uploads → multipart.
  • Slow uploads from far-away users → Transfer Acceleration.
  • Multipart improves speed (parallel) and reliability (retry one part).

Flashcards

Max size for a single PUT?tap
5 GB.
Two benefits of multipart upload?tap
Faster (parallel parts) and more reliable (retry a failed part).
Best feature for remote users uploading big files?tap
S3 Transfer Acceleration.

Key Management Service (KMS)

Create and control encryption keys.

Quick Understanding

ConceptMeaningExam relevance
KMSService to create/manage cryptographic keys.Used by SSE-KMS + many services.
RegionalKeys never leave their region (or KMS).Multi-region needs care.
Envelope encryptionKMS key encrypts a data key; data key encrypts your data.How big data is encrypted.
Key policyResource policy on the key; controls who can use it.Access needs both IAM + key policy.

Key Terms

TermSimple meaning
KMS key (CMK)The main key KMS holds; its material never leaves KMS unencrypted.
Data key (DEK)A key KMS gives you to encrypt large data locally.
AliasA friendly name for a key, e.g. alias/catrobot.
Customer managed keyA key you create and fully control.
Key policyThe policy on the key saying who may use/manage it.

Explanation

graph TD
  CMK[KMS Key - stays in KMS/region] -->|GenerateDataKey| DEK[Data Key]
  DEK --> ENC[Encrypt big data locally]
  CMK -->|encrypt up to 4KB| SMALL[Small data direct]
  POL[Key policy + IAM] --> USE[Who may use the key]
  classDef k fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class CMK k;
  • KMS keys are regional; the key material never leaves KMS unencrypted.
  • KMS can encrypt/decrypt small data (up to 4 KB) directly.
  • For big data: KMS gives a data key (envelope encryption) used locally.
  • Using a key needs permission in both the key policy and the caller's IAM policy.
  • Keys can be backed by FIPS 140-2 validated hardware security modules.

Demo

Demo · CLI v2Situation: the course "battleplans" demo — make a key, encrypt and decrypt a file. Check first: CLI configured as admin.
Create a key + a friendly alias
KID=$(aws kms create-key --description "a4l catrobot" \
  --query KeyMetadata.KeyId --output text)
aws kms create-alias --alias-name alias/catrobot --target-key-id $KID
echo $KID
Expect: a key ID (UUID) and the alias created (no output for alias).
Encrypt a file using the alias
echo "find all the doggos, distract them with the yumz" > battleplans.txt
aws kms encrypt --key-id alias/catrobot \
  --plaintext fileb://battleplans.txt \
  --output text --query CiphertextBlob | base64 --decode > not_battleplans.enc
Expect: a binary file not_battleplans.enc — the encrypted data.
Decrypt it back
aws kms decrypt --ciphertext-blob fileb://not_battleplans.enc \
  --output text --query Plaintext | base64 --decode > decryptedplans.txt
cat decryptedplans.txt
Expect: the original sentence printed — decryption worked.
Fix / Key infoDecrypt doesn't need --key-id: the ciphertext records which key made it. KMS just checks you're allowed to use that key.
Clean upYou can't delete a key instantly — schedule deletion (7–30 days):
aws kms delete-alias --alias-name alias/catrobot
aws kms schedule-key-deletion --key-id $KID --pending-window-in-days 7
rm -f battleplans.txt not_battleplans.enc decryptedplans.txt
Expect: alias removed; key scheduled for deletion in 7 days (you can cancel within the window). A pending key is free.

Build Knowledge

Connects to NOW
  • SSE-KMS (L6) uses these keys; Bucket Keys (L7) cut their cost.
Needed LATER
  • EBS encryption, RDS encryption, Secrets Manager — all use KMS.

Exam Pointers

Exam Tip
  • KMS keys are regional and never leave KMS.
  • KMS directly handles only small data (≤4 KB); bigger data uses data keys (envelope).
  • Access needs key policy + IAM both to allow.
  • Key deletion has a waiting period (7–30 days) — no instant delete.

Flashcards

S3 Object Encryption (CSE / SSE)

Four ways objects can be encrypted at rest.

Quick Understanding

ConceptMeaningExam relevance
CSEClient-Side Encryption: you encrypt before upload.AWS never sees plaintext.
SSE-S3S3 manages keys (AES-256). Simple default.Now the minimum default.
SSE-KMSUses KMS keys → control + audit + role separation.Most flexible. Heavily tested.
SSE-CYou supply the key on each request; S3 does the work.You manage keys yourself.

Key Terms

TermSimple meaning
At restData stored on disk (vs "in transit" = moving over network).
SSEServer-Side Encryption — S3 encrypts for you.
Role separationDifferent people control the key vs the data.

Explanation

MethodWho encryptsWho holds keysUse when…
CSEYou (client)YouAWS must never see plaintext.
SSE-S3S3AWS (AES-256)Simple, hands-off encryption.
SSE-KMSS3 + KMSKMS (you control)Need control, audit, role separation.
SSE-CS3You (provide each request)You want to manage the keys but not the crypto.
  • All S3 objects are now encrypted at rest by default (SSE-S3 at minimum). [added for exam]
  • SSE-KMS role separation: to read an encrypted object you need S3 permission AND KMS key permission. Deny KMS → can't read, even with S3 access.
  • SSE-C: you send the key with every upload/download; lose the key = lose the data.

Demo

Demo · CLI v2Situation: upload objects with SSE-S3 and SSE-KMS, then prove role separation. Check first: CLI as admin; a KMS key (reuse alias alias/catrobot from L5 or make one).
Bucket + an SSE-S3 object
BUCKET=a4l-enc-$RANDOM$RANDOM
aws s3 mb s3://$BUCKET --region us-east-1
echo "plain" > a.txt
aws s3 cp a.txt s3://$BUCKET/a.txt --sse AES256
Expect: upload succeeds; object is encrypted with SSE-S3 (AES256).
An SSE-KMS object
echo "secret" > b.txt
aws s3 cp b.txt s3://$BUCKET/b.txt --sse aws:kms --sse-kms-key-id alias/catrobot
Expect: upload succeeds; this object needs the KMS key to be read.
Prove role separation: deny KMS, then try to read
# Apply this inline policy to a TEST user (not yourself):
# { "Version":"2012-10-17","Statement":[
#   {"Sid":"DenyKMS","Effect":"Deny","Action":"kms:*","Resource":"*"} ] }
# That user can still GET a.txt (SSE-S3) but CANNOT GET b.txt (SSE-KMS).
aws s3 cp s3://$BUCKET/b.txt -   # works for you (you have KMS access)
Expect: with KMS denied, reading the SSE-KMS object returns AccessDenied, while the SSE-S3 object still reads — that's role separation.
Fix / Key infoSSE-KMS ties data access to key access. This is why it's chosen for sensitive data + audit needs.
Clean up
aws s3 rb s3://$BUCKET --force
rm -f a.txt b.txt
Expect: bucket removed. (Remember to also schedule the KMS key for deletion if you're done — see L5.)

Build Knowledge

Connects to NOW
  • Uses KMS (L5). Bucket Keys (L7) reduce SSE-KMS cost.
Needed LATER
  • Default bucket encryption, compliance, KMS across services.

Exam Pointers

Exam Tip
  • SSE-KMS = role separation + audit (CloudTrail logs key use).
  • SSE-C: customer provides the key each request; AWS does the encryption.
  • CSE: AWS never sees plaintext (you encrypt before sending).
  • "Read object needs key access too" → that's SSE-KMS.
Easy to confuseSSE-C (you give the key) vs SSE-KMS (KMS holds the key) vs CSE (you encrypt yourself before upload).

Flashcards

In SSE-C, who provides the key?tap

S3 Bucket Keys

Cut SSE-KMS costs at high volume.

Quick Understanding

ConceptMeaningExam relevance
ProblemSSE-KMS calls KMS for every object → many requests = cost + throttling.Why bucket keys exist.
Bucket KeyA bucket-level key reduces calls to KMS.Up to 99% fewer KMS requests.

Key Terms

TermSimple meaning
KMS request costYou pay per KMS API call (encrypt/decrypt/generate).
ThrottlingBeing slowed down when you make too many requests.
Bucket KeyA shared key at the bucket level so S3 calls KMS far less.

Explanation

graph TD
  WO[Without bucket key] --> MANY[1 KMS call per object = $$ + throttle]
  WK[With bucket key] --> FEW[Few KMS calls, S3 reuses a bucket-level key]
  FEW --> SAVE[Up to 99% fewer KMS requests]
  classDef g fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class WK g;
  • Only relevant to SSE-KMS at high object volume.
  • Reduces KMS API requests (and cost) by up to 99%, and lowers throttling risk.
  • Enabled per bucket (or per object on upload).

Demo

S3 Access Points & MRAP

Simpler access management for shared data; global routing.

Quick Understanding

ConceptMeaningExam relevance
Access PointA named hostname with its own policy + network rules for a bucket.Manage access at scale.
Per-app accessGive each app/team its own access point + policy.No giant bucket policy.
MRAPMulti-Region Access Point: one global endpoint over many buckets.Global routing + failover.

Key Terms

TermSimple meaning
Access pointA custom entry point to a bucket with its own permissions.
Access point policyPermissions just for that access point.
VPC restrictionAn access point can be locked to a specific VPC.
MRAPA single global endpoint that routes to buckets in several regions.

  • An access point = a unique hostname + its own policy + optional VPC-only rule.
  • Instead of one huge bucket policy, give each app/team a tailored access point.
  • MRAP gives one global endpoint that routes requests to buckets across regions (latency-based, with failover).

Demo

Demo · CLI v2Situation: create an access point on a bucket, then remove it. Check first: CLI as admin; know your account ID.
Bucket + access point
ACC=$(aws sts get-caller-identity --query Account --output text)
BUCKET=a4l-ap-$RANDOM$RANDOM
aws s3 mb s3://$BUCKET --region us-east-1
aws s3control create-access-point --account-id $ACC \
  --name team-a --bucket $BUCKET
Expect: JSON with the access point ARN/alias. Apps can now use this access point instead of the bucket directly.
List access points on the bucket
aws s3control list-access-points --account-id $ACC --bucket $BUCKET \
  --query "AccessPointList[].Name"
Expect: ["team-a"].
Fix / Key infoEach access point has its own policy; you can also force it to be reachable only from a VPC — great for isolating teams/apps.
Clean up
aws s3control delete-access-point --account-id $ACC --name team-a
aws s3 rb s3://$BUCKET --force
Expect: access point + bucket removed. (MRAP is best explored via the course's GitHub mini-project.)

Mock Exam

Exam-style questions over S3 + IAM / Accounts / Organisations.
How to useAnswer in your head first, then tap Show answer. Aim for the best option, not just a correct one. 14 questions.
1. A bucket policy allows public read, but objects return Access Denied. What's the most likely cause?
A) Wrong storage classB) Block Public Access is enabledC) Versioning is offD) No lifecycle rule
B. Block Public Access overrides any policy that grants public access.
2. An EC2 app must read objects from S3. What is best practice?
A) Store IAM user access keys on the instanceB) Attach an IAM role to the instanceC) Use the root user keysD) Make the bucket public
B. Instance roles give temporary, rotating credentials — no stored keys to leak.
3. You must keep audit records un-deletable for 3 years, with NO exceptions even for admins. Which feature?
A) VersioningB) Object Lock — GovernanceC) Object Lock — ComplianceD) Lifecycle expiration
C. Compliance mode prevents deletion until the period ends, even for root.
4. A user has an Allow for s3:* but an SCP on their account denies s3:*. Can they use S3?
A) Yes, identity Allow winsB) No, the SCP boundary blocks itC) Only readsD) Only in us-east-1
B. Action must be allowed by both the SCP and the identity policy; the SCP deny wins.
5. Cheapest storage for backups kept 10 years, accessed maybe once, restore in 12+ hours is fine.
A) S3 StandardB) Standard-IAC) Glacier Deep ArchiveD) One Zone-IA
C. Deep Archive is cheapest for rarely-read, slow-restore long-term data.
6. Give a partner in another AWS account read access to your bucket. Best method?
A) ACLB) Bucket policy naming their accountC) Your identity policyD) Pre-signed URL forever
B. Bucket policies handle cross-account access; identity policies can't grant to other accounts' principals directly.
7. Replication was enabled, but files uploaded before it didn't copy. Why?
A) Replication isn't retroactiveB) Wrong regionC) ACL blocked itD) No lifecycle
A. Only new objects replicate; use Batch Replication for existing ones.
8. Millions of mobile app users need temporary, limited AWS access. Best design?
A) One IAM user eachB) Web identity federation + roleC) Shared access keyD) Root credentials
B. IAM users cap at 5000; federation (e.g. Cognito) + roles scales to millions.
9. Share a single private file with a customer who has no AWS account, for 1 hour.
A) Make bucket publicB) Pre-signed URLC) New IAM userD) Access point
B. A pre-signed URL gives time-limited access to one object, no login.
10. A regulator audit asks "who deleted this IAM role last month?" Where do you look?
A) CloudWatch metricsB) CloudTrailC) S3 access logsD) Cost Explorer
B. CloudTrail logs API calls and account events — who did what.
11. SSE-KMS bucket with millions of objects has huge KMS costs. Keep SSE-KMS but cut cost?
A) Switch to SSE-CB) Enable S3 Bucket KeysC) Turn off encryptionD) Use One Zone-IA
B. Bucket Keys cut KMS requests up to 99% with no security change.
12. Restrict ALL member accounts in an org to only the eu-west-1 region. Best tool?
A) IAM groupB) SCP at the org root/OUC) Bucket policyD) Pre-signed URL
B. An SCP sets an org-wide boundary (note: the management account is not affected).
13. You need an S3 static website served over HTTPS with a custom domain. What do you add?
A) Nothing, S3 supports HTTPSB) CloudFront in front of S3C) Transfer AccelerationD) MRAP
B. The S3 website endpoint is HTTP only; CloudFront adds HTTPS + custom domain.
14. A user has S3 read but can't open SSE-KMS objects. Quickest correct fix?
A) Enable versioningB) Grant the user access to the KMS keyC) Make the object publicD) Add a lifecycle rule
B. SSE-KMS uses role separation — reading needs KMS key permission too.
