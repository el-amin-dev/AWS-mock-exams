Public vs Private Services

"Public" means where it sits on the network — not who can use it.

Quick Understanding

ConceptMeaningExam relevance
Public serviceSits in the AWS Public Zone; reachable from the internet (but still needs permission).Common trap: "public" ≠ "open to everyone".
Private serviceRuns inside a VPC (private network).Needs network access into the VPC.
3 network zonesPublic Internet · AWS Public Zone · AWS Private Zone (VPC).Mental model for all networking.

Key Terms

TermSimple meaning
VPCVirtual Private Cloud — your own private network inside AWS.
AWS Public ZoneAWS-run network that sits between the internet and AWS; public services attach here (e.g. S3).
PermissionsRules that decide who is allowed to use a service, separate from the network.

Explanation

graph LR
  NET[Public Internet] --> PUB[AWS Public Zone]
  PUB --> S3[S3 - a public service]
  PUB -. gateway .-> VPC[AWS Private Zone - VPC]
  VPC --> EC2[EC2 - a private service]
  classDef p fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  classDef v fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class PUB,S3 p; class VPC,EC2 v;
  • Public service (e.g. S3): attaches to the AWS Public Zone, reachable over the internet — but access is still controlled by permissions.
  • Private service (e.g. EC2): lives inside a VPC and is private by default.
  • "Public" describes the network position, not whether the public can read your data.

Demo

Demo · CLI v2 (read-only)Situation: prove a public service (S3) is reachable straight over the internet from your laptop. Check first: CLI configured (aws sts get-caller-identity works).
Talk to S3 (public service) over the internet
aws s3 ls
Expect: a list of your buckets (or empty). No VPC needed — that's a public service.
Fix / Key infoReaching S3 still required valid credentials. "Public" got you to the door; permissions opened it.
Clean upNothing created — read-only. No cost.

Exam Tip

  • Private services need a path into the VPC (gateway, VPN, endpoint).

AWS Global Infrastructure

Regions, Availability Zones, Edge Locations + resilience levels.

Quick Understanding

ConceptMeaningExam relevance
RegionA separate geographic area (e.g. us-east-1) with multiple AZs.Isolation, data sovereignty, latency.
Availability Zone (AZ)One or more isolated data centres inside a region.Spread across AZs = high availability.
Edge LocationSmall site near users for caching/delivery (CDN).CloudFront / low-latency delivery.
Resilience levelsGlobally / Region / AZ resilient.Know which services are which.

Key Terms

TermSimple meaning
Region codeShort name like us-east-1 (location) — the "1" is just an ID.
AZ nameRegion code + letter, e.g. us-east-1a.
ResilientAble to keep working when part of it fails.
LatencyDelay/time for data to travel. Lower = faster.

Explanation

graph TD
  GLOBE[AWS Global Network] --> R1[Region: us-east-1]
  GLOBE --> R2[Region: eu-west-1]
  R1 --> AZ1[AZ us-east-1a]
  R1 --> AZ2[AZ us-east-1b]
  R1 --> AZ3[AZ us-east-1c]
  GLOBE --> EDGE[Many Edge Locations
near users]
  classDef g fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class GLOBE g;
LevelSurvivesExample services
Globally resilientLoss of a whole regionIAM, Route 53
Region resilientLoss of an AZ (data copied across AZs)S3, DynamoDB
AZ resilientHardware failures within that one AZ onlyEBS volume, a single EC2
  • A region has multiple AZs (usually 3+), physically separated but close.
  • Put copies in different AZs for high availability inside a region.
  • Use multiple regions for global reach and disaster recovery.

Demo

Demo · CLI v2 (read-only)Situation: list AWS regions and the AZs in one region. Check first: CLI configured.
List all regions
aws ec2 describe-regions --query "Regions[].RegionName" --output table
Expect: a table of region codes (us-east-1, eu-west-1, …).
List AZs in one region
aws ec2 describe-availability-zones \
  --region us-east-1 \
  --query "AvailabilityZones[].ZoneName" --output table
Expect: us-east-1a, us-east-1b, … — the AZs of that region.
Fix / Key infoAZ letters (a, b, c) are mapped differently per account, so "us-east-1a" may be a different physical AZ for someone else.
Clean upRead-only. Nothing to delete, no cost.

Build Knowledge

Connects to NOW
  • Default VPC (Lesson 3) puts one subnet in each AZ.
  • EC2 (Lesson 4) lives in one AZ.
Needed LATER
  • Multi-AZ RDS, Auto Scaling, Load Balancers, and DR strategies all rely on AZs/regions.

Exam Pointers

Exam Tip
  • Need to survive a whole region failing → think multi-region / globally resilient services.
  • Need cheap HA inside one region → spread across AZs.
  • IAM & Route 53 are global; CloudFront uses edge locations; most other services are regional.
Easy to confuseRegion = big geographic area. AZ = data centre(s) inside it. Edge location = tiny cache site for fast delivery, not for running servers.

IAM and Route 53.

Caching/delivering content close to users (low latency, e.g. CloudFront).

Default VPC

The ready-made network in each region.

Quick Understanding

ConceptMeaningExam relevance
Default VPCA pre-built network, one per region, made automatically.Used in beginner demos.
Fixed CIDRAlways uses IP range 172.31.0.0/16.Remember the range.
1 subnet per AZA default subnet in each AZ, sized /20.Links to AZ concept.
Internet-readyHas internet gateway + auto public IPs.Why default instances reach the net.

Key Terms

TermSimple meaning
CIDRA way to write a block of IP addresses, e.g. 172.31.0.0/16.
SubnetA smaller slice of the VPC's IP range, placed in one AZ.
Internet Gateway (IGW)The door that lets the VPC talk to the internet.
Route tableRules that say where network traffic should go.

Explanation

graph TD
  DV[Default VPC 172.31.0.0/16] --> IGW[Internet Gateway]
  DV --> SA[Subnet /20 in AZ-a]
  DV --> SB[Subnet /20 in AZ-b]
  DV --> SC[Subnet /20 in AZ-c]
  SA --> PUBIP[Auto-assign public IPs]
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class DV k;
  • Exactly one default VPC per region; you can delete and recreate it from the console.
  • Always the same range 172.31.0.0/16 and one /20 subnet per AZ.
  • Comes with IGW, route table, default security group and NACL, and auto public IPs.
  • Fine for quick tests; custom VPCs are best for real/production work.

Demo

Demo · CLI v2 (read-only)Situation: inspect your region's default VPC and its subnets. Check first: CLI configured.
Find the default VPC + its CIDR
aws ec2 describe-vpcs \
  --filters "Name=isDefault,Values=true" \
  --query "Vpcs[].{VpcId:VpcId,Cidr:CidrBlock}" --output table
Expect: one VPC with CIDR 172.31.0.0/16.
List its default subnets (one per AZ)
aws ec2 describe-subnets \
  --filters "Name=defaultForAz,Values=true" \
  --query "Subnets[].{AZ:AvailabilityZone,Cidr:CidrBlock}" --output table
Expect: one /20 subnet per AZ (e.g. 172.31.0.0/20, 172.31.16.0/20 …).
Fix / Key infoIf a region has no default VPC (someone deleted it), you can recreate it: aws ec2 create-default-vpc.
Clean upRead-only — nothing to delete. Don't delete your default VPC if early demos rely on it.

Build Knowledge

Connects to NOW
  • EC2 (4), CloudFormation (6) and CloudWatch (7) demos launch into this VPC.
Needed LATER
  • The big VPC section (custom CIDR, public/private subnets, NAT) replaces this default approach.

Exam Pointers

Exam Tip
  • Default VPC CIDR = 172.31.0.0/16 (memorise).
  • One per region, deletable and recreatable.
  • Default subnets auto-assign public IPs — that's why default instances reach the internet.

Flashcards

172.31.0.0/16.
How many default VPCs per region?tap
One. It can be deleted and recreated.

EC2 Basics

Virtual servers you rent — called instances.

Quick Understanding

ConceptMeaningExam relevance
EC2 = IaaSYou rent virtual machines (instances) and manage the OS.Core compute service.
AMIA template (OS + config) used to launch instances.Where instances come from.
Instance statespending → running → stopping/stopped → terminated.Billing differs per state.
ConnectLinux = SSH (22) + key pair; Windows = RDP (3389).Ports + key pairs tested.

Key Terms

TermSimple meaning
InstanceOne running virtual machine.
AMIAmazon Machine Image — the "recipe" to build an instance.
Instance typeThe size/power, e.g. t2.micro (free-tier friendly).
Key pairPublic key on the instance + private key you keep, used for SSH login.
Security groupA virtual firewall around the instance.

Explanation

stateDiagram-v2
  [*] --> pending
  pending --> running
  running --> stopping
  stopping --> stopped
  stopped --> pending: start
  running --> shutting_down
  stopped --> shutting_down: terminate
  shutting_down --> terminated
  terminated --> [*]
  • An instance runs in one subnet, in one AZ — it is AZ-resilient only.
  • running = you pay for compute. stopped = no compute charge, but you still pay for its EBS storage. terminated = gone.
  • Launch from an AMI; choose an instance type, key pair, and security group.
  • Connect to Linux with SSH (port 22) using your private key.

Demo

Demo · CLI v2Situation: launch a tiny Animals4life test server, then remove it. Check first: CLI configured; default VPC exists; use a free-tier type.
Create an SSH key pair (saved locally)
aws ec2 create-key-pair --key-name a4l-key \
  --query "KeyMaterial" --output text > a4l-key.pem
chmod 400 a4l-key.pem
Expect: a file a4l-key.pem created. Keep it safe — it's your only copy of the private key.
Get the latest Amazon Linux 2023 AMI ID
AMI=$(aws ssm get-parameters \
  --names /aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64 \
  --query "Parameters[0].Value" --output text)
echo $AMI
Expect: an AMI ID like ami-0abc123… stored in $AMI.
Launch one t2.micro instance
aws ec2 run-instances \
  --image-id $AMI \
  --instance-type t2.micro \
  --key-name a4l-key \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=a4l-test}]'
Expect: JSON with an InstanceId (e.g. i-0123…) and state pending.
Check it is running + get its public IP
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=a4l-test" \
  --query "Reservations[].Instances[].{State:State.Name,IP:PublicIpAddress,Id:InstanceId}" \
  --output table
Expect: State running and a public IP. (Connect with ssh -i a4l-key.pem ec2-user@PUBLIC_IP if your default security group allows port 22.)
Fix / Key info
  • Use t2.micro (or t3.micro) to stay in free tier. Bigger types cost money.
  • If SSH fails, your security group probably blocks port 22 from your IP.
  • Stopping still bills the EBS volume; terminate to stop all charges.
Clean upTerminate the instance and delete the key:
ID=$(aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=a4l-test" "Name=instance-state-name,Values=running,stopped,pending" \
  --query "Reservations[].Instances[].InstanceId" --output text)
aws ec2 terminate-instances --instance-ids $ID
aws ec2 wait instance-terminated --instance-ids $ID
aws ec2 delete-key-pair --key-name a4l-key
rm -f a4l-key.pem
Expect: instance moves to terminated; key pair removed. No more charges.

Build Knowledge

Connects to NOW
  • Launches into the default VPC (Lesson 3).
  • CloudWatch (7) monitors this instance; CloudFormation (6) can build it for you.
Needed LATER
  • Instance types, EBS, AMIs, auto-scaling, instance roles — a whole EC2 section builds on this.

Exam Pointers

Exam Tip
  • stopped instances still cost EBS storage; only terminate stops all cost.
  • EC2 lives in one AZ → not highly available on its own.
  • SSH = port 22 (Linux); RDP = port 3389 (Windows).
  • You keep the private key; the public key sits on the instance.
Trap"Stop" ≠ "Terminate". Stop = pause (data on EBS kept, EBS billed). Terminate = delete (root volume usually deleted too).

Flashcards

What is an AMI?tap
A template (OS + config) used to launch EC2 instances.
Are you billed for a stopped instance?tap
No compute charge, but yes for the attached EBS storage.
Which port for SSH to Linux?tap
Port 22.
How many AZs does one instance span?tap
One — it runs in a single AZ.

S3 Basics

Object storage that scales almost without limit.

Quick Understanding

ConceptMeaningExam relevance
Object storageStore whole files ("objects") in "buckets". Not a disk, not a file share.Know what S3 is / isn't.
Bucket nameMust be globally unique, lowercase, DNS-style.Common detail question.
Region-basedA bucket is created in one region.Data location matters.
Private by defaultBlock Public Access is ON when created.Security default.

Key Terms

TermSimple meaning
ObjectOne stored item = key (name) + value (data) + metadata.
BucketA container that holds objects.
Prefix ("folder")Folders aren't real — they're just part of the object's key/name.
ARNUnique ID for the bucket, e.g. arn:aws:s3:::my-bucket.
Block Public AccessA safety switch that keeps buckets/objects private.

Explanation

graph TD
  REG[Region] --> B[Bucket - globally unique name]
  B --> O1[Object key: photos/cat.jpg]
  B --> O2[Object key: report.pdf]
  O1 --> V[value = file data]
  O1 --> M[metadata]
  classDef k fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class B k;
  • An object = a key (its name) + the data + metadata. Size 0 bytes to 5 TB.
  • Bucket names are globally unique across all AWS accounts, 3–63 chars, lowercase.
  • "Folders" are just prefixes in the key — there are no real folders.
  • Buckets are private by default (Block Public Access on).
  • S3 is not a disk you mount and not a file share — it's object storage.

Demo

Demo · CLI v2Situation: store an Animals4life file in S3, then remove it. Check first: CLI configured. Pick a unique bucket name (add random digits).
Create a uniquely-named bucket
BUCKET=a4l-demo-$RANDOM$RANDOM
aws s3 mb s3://$BUCKET --region us-east-1
echo $BUCKET
Expect: make_bucket: a4l-demo-XXXX. If the name is taken, run again (names are global).
Upload an object
echo "hello animals4life" > cat.txt
aws s3 cp cat.txt s3://$BUCKET/photos/cat.txt
Expect: upload: ./cat.txt to s3://…/photos/cat.txt. ("photos/" is just a prefix, not a real folder.)
List what's inside
aws s3 ls s3://$BUCKET --recursive
Expect: one line showing photos/cat.txt with size and date.
Fix / Key info
  • The full object name (key) is photos/cat.txt — the "folder" is part of the name.
  • By default the object is private; nobody can read it without permission.
Clean upEmpty then delete the bucket (a bucket must be empty to delete):
aws s3 rm s3://$BUCKET --recursive
aws s3 rb s3://$BUCKET
rm -f cat.txt
Expect: objects deleted, then remove_bucket: a4l-demo-XXXX. No cost left.

Build Knowledge

Connects to NOW
  • S3 is a public service (Lesson 1) — reached over the internet.
  • CloudFormation templates and logs often land in S3.
Needed LATER
  • Storage classes, versioning, encryption, static hosting, replication — the big S3 section.

Exam Pointers

Exam Tip
  • Bucket names are globally unique (across all accounts), lowercase, DNS-style.
  • Objects up to 5 TB; storage is unlimited in practice.
  • S3 is object storage — you can't mount it as an OS disk (that's EBS/EFS).
  • Private by default; "make it public" means changing Block Public Access + permissions.
Easy to confuseS3 (object) vs EBS (block disk for one EC2) vs EFS (shared file system). If the question says "mount as a drive", it's not S3.

Build, change and delete infrastructure from code.

Quick Understanding

ConceptMeaningExam relevance
IaCInfrastructure as Code — describe resources in a file, AWS builds them.Automation/repeatability.
TemplateA YAML or JSON file listing what to create.Format options tested.
StackThe live group of resources made from a template.Lifecycle unit.
Logical vs physicalName in template (logical) → real resource (physical).Wording trap.

Key Terms

TermSimple meaning
YAML / JSONTwo text formats for writing templates.
ResourceOne thing to create (an EC2, a bucket…). The only required template part.
Logical resourceThe name you give a resource inside the template.
Physical resourceThe real AWS thing created (with a real ID).

Explanation

graph LR
  T[Template YAML/JSON] --> S[Stack]
  S --> R1[Physical resource: EC2]
  S --> R2[Physical resource: SG]
  T -. defines .-> L[Logical resources]
  L -. become .-> R1
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class S k;
  • Write a template (YAML/JSON) → CloudFormation makes a stack → the stack creates resources.
  • Update the template → the stack updates only what changed.
  • Delete the stack → it removes all its resources (great for clean-up).
  • Only the Resources section is required in a template.

Demo

Demo · CLI v2 + course templateSituation: use the course's template to create an EC2 instance as a stack, then delete it all in one step. Check first: CLI configured; default VPC exists.
Download the course template
curl -o ec2instance.yaml \
  https://learn-cantrill-labs.s3.amazonaws.com/awscoursedemos/0058-aws-simplecfn/ec2instance.yaml
head -20 ec2instance.yaml

Create the stack
aws cloudformation create-stack \
  --stack-name a4l-cfn-demo \
  --template-body file://ec2instance.yaml
Expect: JSON with a StackId. Creation has started.
Wait until it finishes, then list resources
aws cloudformation wait stack-create-complete --stack-name a4l-cfn-demo
aws cloudformation describe-stack-resources --stack-name a4l-cfn-demo \
  --query "StackResources[].{Logical:LogicalResourceId,Physical:PhysicalResourceId,Status:ResourceStatus}" \

Expect: a table showing the logical name (from the template) next to the real physical ID (e.g. i-0abc…).
Fix / Key info
  • The instance was created by the stack, not by you directly — so let the stack manage it.
  • If create fails, run describe-stack-events to see the first error.
Clean upDeleting the stack deletes everything it made — no leftovers:
aws cloudformation delete-stack --stack-name a4l-cfn-demo
aws cloudformation wait stack-delete-complete --stack-name a4l-cfn-demo
rm -f ec2instance.yaml
Expect: the stack and its EC2 instance are removed. No running resources, no cost.

CloudWatch Basics

Metrics, logs and alarms for your AWS resources.

Quick Understanding

ConceptMeaningExam relevance
MetricsNumbers over time (e.g. CPU %).What you alarm on.
LogsText log lines collected centrally (via the agent).Needs CloudWatch Agent on EC2.
AlarmWatches a metric and changes state when a threshold is crossed.States are tested.
Agent for memoryMemory/disk metrics need the CloudWatch Agent.Classic trap.

Key Terms

TermSimple meaning
NamespaceA folder for related metrics, e.g. AWS/EC2.
DimensionA label that picks one item, e.g. a specific InstanceId.
DatapointOne measured value at a time.
Alarm stateOK · ALARM · INSUFFICIENT_DATA.
CloudWatch AgentSoftware on the instance to send OS-level metrics + logs.

Explanation

graph TD
  EC2[EC2 instance] -->|CPU metric| CW[CloudWatch]
  AGENT[CloudWatch Agent] -->|memory, disk, logs| CW
  CW --> AL{Alarm threshold}
  AL -->|below| OK[State: OK]
  AL -->|above| ALARM[State: ALARM]
  ALARM --> ACT[Action: notify / scale]
  classDef k fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class CW k;
  • CloudWatch collects metrics, logs and events from AWS services.
  • Basic EC2 metrics (CPU, network) come for free; memory + disk + logs need the agent.
  • An alarm compares a metric to a threshold and sits in OK / ALARM / INSUFFICIENT_DATA.
  • Standard monitoring = every 5 min; detailed monitoring = every 1 min (extra cost).

Demo

Demo · CLI v2Situation: alarm when an Animals4life instance's CPU goes above 15%, then push CPU up with stress to trigger it. Check first: you have a running t2.micro (reuse the Lesson 4 demo) and its InstanceId.
On the instance: install and run stress
# run these INSIDE the instance over SSH
sudo dnf install -y stress    # (Amazon Linux 2 uses: sudo yum install stress -y)
stress -c 2
Expect: stress keeps running and loads 2 CPU cores. Leave it running for a few minutes.
From your laptop: create the CPU alarm
aws cloudwatch put-metric-alarm \
  --alarm-name a4l-cpu-high \
  --namespace AWS/EC2 \
  --metric-name CPUUtilization \
  --dimensions Name=InstanceId,Value=i-REPLACE_ME \
  --statistic Average --period 60 --evaluation-periods 1 \
  --threshold 15 --comparison-operator GreaterThanThreshold
Expect: no output = alarm created.
Watch the alarm move to ALARM
aws cloudwatch describe-alarms --alarm-names a4l-cpu-high \
  --query "MetricAlarms[].StateValue" --output text
Expect: first OK or INSUFFICIENT_DATA, then ALARM once CPU passes 15%. Press Ctrl+C on stress → it returns to OK.
Fix / Key info
  • CPU is a built-in EC2 metric — no agent needed. Memory is not; that would need the agent.
  • A new alarm starts as INSUFFICIENT_DATA until enough datapoints arrive.
Clean upDelete the alarm (and terminate the instance from the Lesson 4 clean-up):
aws cloudwatch delete-alarms --alarm-names a4l-cpu-high
Expect: no output = alarm deleted. Then terminate the EC2 to stop all charges.

TermSimple meaning

HA vs FT vs DR

Three resilience words people mix up.

Quick Understanding

ConceptMeaningExam relevance
High Availability (HA)Minimise downtime; recover fast (short outage allowed).Most-tested distinction.
Fault Tolerance (FT)No downtime; keep working through failure.More costly, full redundancy.
Disaster Recovery (DR)Plan to recover after a big failure.RPO / RTO.

Key Terms

TermSimple meaning
DowntimeTime the system is not working.
RedundancySpare copies that take over if one fails.
FailoverSwitching to a backup when the main thing fails.
RPORecovery Point Objective — how much data loss you can accept. [added for exam]
RTORecovery Time Objective — how long recovery may take. [added for exam]

Explanation

graph LR
  F[Something fails] --> HA[HA: quick failover
brief outage OK]
  F --> FT[FT: keeps running
zero interruption]
  F --> DR[DR: restore from
backups / other region]
  classDef k fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class HA,FT,DR k;
  • HA = aim for maximum uptime. A short outage + fast recovery is acceptable (users may need to retry/re-login).
  • FT = the system keeps working with no interruption, even during a failure. Needs full redundancy → more expensive.
  • DR = your plan + tools to recover after a serious failure (measured by RPO and RTO).

Exam Pointers

Exam Tip
  • Keyword "minimise downtime / fast recovery" → HA.
  • Keyword "no interruption / always on" → FT (and it costs more).
  • Keyword "recover after disaster / backups / RPO / RTO" → DR.
Easy to confuseHA is not FT. HA accepts a brief outage during failover; FT allows none. FT is harder and pricier.

Key Terms

TermSimple meaning
DNSThe internet's phone book — turns names into IP addresses.
Hosted zoneWhere a domain's DNS records live in Route 53.

DNS Record Types

A, AAAA, CNAME, MX, TXT, NS + TTL.

Quick Understanding

ConceptMeaningExam relevance
Record typesDifferent records do different jobs in DNS.Match type to need.
CNAME limitA CNAME can't sit at the zone apex (naked domain).Classic trap → use Alias.
TTLHow long a resolver caches a record.Affects change speed.

Key Terms

TermSimple meaning
ResolverThe service that looks up DNS for you (often your ISP).
Apex / naked domainThe bare domain with no prefix, e.g. animals4life.org.
TTLTime To Live — seconds a record stays cached.
Alias recordAWS-only record that works at the apex (use instead of CNAME). [added for exam]

Explanation

RecordMaps…Example use
Aname → IPv4 addresswww → 1.2.3.4
AAAAname → IPv6 addresswww → 2606:…
CNAMEname → another namewww → app.example.com
MXdomain → mail servers (+priority)email delivery
TXTdomain → text datadomain verification, SPF
NSzone → its nameserversdelegation between zones
graph TD
  Q[Lookup www.animals4life.org] --> A[A record -> IPv4]
  Q --> AAAA[AAAA record -> IPv6]
  C[CNAME] --> OTHER[points to another NAME, not an IP]
  TTL[TTL] --> CACHE[how long resolvers cache the answer]
  classDef k fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class A,AAAA k;
  • A = IPv4, AAAA = IPv6 (note the double-A is the "bigger" address).
  • CNAME points a name to another name — never directly to an IP, and not at the apex.
  • MX carries a priority number — lower number = higher priority.
  • TTL high = fewer lookups but slower to change; low = faster changes, more lookups.
