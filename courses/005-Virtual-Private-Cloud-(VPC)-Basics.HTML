VPC Sizing & Structure

Plan your IP address space before building.

Quick Understanding

ConceptMeaningExam relevance
VPC CIDRThe IP range a VPC uses (e.g. 10.16.0.0/16).Pick once; hard to change.
Private rangesRFC1918: 10.x, 172.16–31.x, 192.168.x.Use these inside a VPC.
Avoid overlapsDon't clash with other VPCs, on-prem, other clouds.Needed for peering/VPN.
Size limitsVPC CIDR: min /28, max /16.Common number question.

Key Terms

TermSimple meaning
CIDRA way to write an IP range, e.g. 10.16.0.0/16.
/16A big range = 65,536 addresses. Smaller number = bigger range.
RFC1918The standard private IP ranges, not used on the public internet.
OverlapTwo networks sharing the same IPs — breaks connections between them.

Explanation

graph TD
  PLAN[Plan IP space first] --> R[Use private RFC1918 ranges]
  R --> AVOID[Avoid common ranges
e.g. 192.168.x, 10.0.x]
  AVOID --> NOOVERLAP[No overlap with on-prem / other VPCs / clouds]
  NOOVERLAP --> SIZE[Size per VPC: a /16 is a safe default]
  SIZE --> ROOM[Leave room: regions x accounts x AZs x growth]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class PLAN a;
  • Pick private ranges (RFC1918). Avoid ranges others commonly use, so you can peer/VPN later.
  • VPC CIDR can be /28 (16 IPs) to /16 (65,536 IPs). You can add secondary CIDRs later, but resizing the primary is painful — plan well.
  • Reserve enough space for multiple regions, accounts, AZs and future growth.
  • Animals4life plan: each VPC gets its own /16 (e.g. 10.16.0.0/16 = US region 1, General account, VPC1), starting at 10.16 to dodge common ranges.
CIDR size cheat
  • /16 = 65,536 IPs · /20 = 4,096 · /24 = 256 · /28 = 16.
  • Each +1 to the number halves the range; each −1 doubles it.

Custom VPCs

Your own isolated network in a region.

Quick Understanding

ConceptMeaningExam relevance
VPCA private, isolated network you control.Foundation of everything.
RegionalA VPC spans all AZs in ONE region.Not global, not one-AZ.
DNS optionsenableDnsSupport + enableDnsHostnames.Needed for public DNS names.
TenancyDefault (shared) vs Dedicated (own hardware, costly).Pick default unless required.

Key Terms

TermSimple meaning
Custom VPCA VPC you create and configure (vs the default one).
Secondary CIDRExtra IP ranges you can add to a VPC.
enableDnsSupportLets resources resolve DNS names inside the VPC.
enableDnsHostnamesGives instances with public IPs a public DNS name.
TenancyWhether your instances share hardware (default) or not (dedicated).

Explanation

graph TD
  REGION[AWS Region] --> VPC[Custom VPC 10.16.0.0/16]
  VPC --> AZA[AZ-a]
  VPC --> AZB[AZ-b]
  VPC --> AZC[AZ-c]
  VPC --> DNS[DNS: support + hostnames]
  VPC --> IPV6[Optional IPv6 /56 from Amazon]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class VPC a;
  • A VPC is regional — it covers every AZ in that region.
  • It has a primary IPv4 CIDR (and optional secondary CIDRs), plus an optional IPv6 /56.
  • For instances to get public DNS names, enable DNS support + DNS hostnames.
  • Keep default tenancy unless a rule forces dedicated hardware (dedicated is expensive).

Demo

Demo · CLI v2Situation: build the A4L VPC shell (10.16.0.0/16) with DNS on. Check first: CLI configured; region us-east-1.
Create the VPC
VPC=$(aws ec2 create-vpc --cidr-block 10.16.0.0/16 \
  --tag-specifications 'ResourceType=vpc,Tags=[{Key=Name,Value=a4l-vpc}]' \
  --query Vpc.VpcId --output text)
echo $VPC
Expect: a VPC ID like vpc-0abc.... Save it.
Turn on DNS support + hostnames
aws ec2 modify-vpc-attribute --vpc-id $VPC --enable-dns-support '{"Value":true}'
aws ec2 modify-vpc-attribute --vpc-id $VPC --enable-dns-hostnames '{"Value":true}'
Expect: no output = both enabled.
Confirm
aws ec2 describe-vpcs --vpc-ids $VPC \
  --query "Vpcs[0].{Cidr:CidrBlock,State:State}"
Expect: CIDR 10.16.0.0/16, state available.
Fix / Key infoKeep $VPC for the next lessons — subnets, gateways and routes all attach to it. (Creating a VPC is free.)
Clean upWait until the end of Lesson 8 (everything builds on this VPC). To delete now:
aws ec2 delete-vpc --vpc-id $VPC
Expect: no output (only works once subnets/gateways are removed).

VPC Subnets

Split the VPC into AZ-based pieces.

Quick Understanding

ConceptMeaningExam relevance
SubnetA slice of the VPC CIDR, living in ONE AZ.Subnet = one AZ. Key fact.
Reserved IPs5 IPs per subnet are reserved by AWS.Very common question.
Auto-assign IPPer-subnet switch to give instances a public IPv4.Makes a subnet "public".
IPv6Each subnet gets a /64 from the VPC /56.Detail question.

Key Terms

TermSimple meaning
AZAvailability Zone — one isolated data-center area in a region.
Reserved IPAn address AWS keeps for itself in each subnet.
Auto-assign public IPv4Subnet option: new instances get a public IP automatically.
/64The IPv6 size each subnet uses.

Explanation

graph TD
  VPC[VPC 10.16.0.0/16] --> A[Subnet web-A 10.16.48.0/20 - AZ a]
  VPC --> B[Subnet app-A 10.16.32.0/20 - AZ a]
  VPC --> C[Subnet db-A 10.16.16.0/20 - AZ a]
  A --> R[5 IPs reserved per subnet]
  classDef a fill:#0f3b39,stroke:#2dd4bf,color:#e7edf3;
  class VPC a;
The 5 reserved IPs (in e.g. 10.16.16.0/24)
  • .0 network address
  • .1 VPC router
  • .2 DNS (and AWS services)
  • .3 reserved for future use
  • .255 network broadcast (not supported, but reserved)
  • A subnet lives in one AZ and uses a sub-range of the VPC CIDR.
  • 5 IPs are reserved per subnet, so usable = total − 5.
  • Set auto-assign public IPv4 on a subnet to make instances reachable (with routing).
  • If the VPC has IPv6, each subnet takes a /64.

Demo

Demo · CLI v2Situation: create A4L web/app/db subnets in AZ-a. Check first: $VPC from Lesson 2 still set.
Create three subnets in AZ-a
AZ=us-east-1a
WEBA=$(aws ec2 create-subnet --vpc-id $VPC --cidr-block 10.16.48.0/20 \
  --availability-zone $AZ \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=sn-web-A}]' \
  --query Subnet.SubnetId --output text)
APPA=$(aws ec2 create-subnet --vpc-id $VPC --cidr-block 10.16.32.0/20 \
  --availability-zone $AZ \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=sn-app-A}]' \
  --query Subnet.SubnetId --output text)
DBA=$(aws ec2 create-subnet --vpc-id $VPC --cidr-block 10.16.16.0/20 \
  --availability-zone $AZ \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=sn-db-A}]' \
  --query Subnet.SubnetId --output text)
echo "web=$WEBA app=$APPA db=$DBA"
Expect: three subnet IDs printed.
Make the web subnet auto-assign public IPv4
aws ec2 modify-subnet-attribute --subnet-id $WEBA --map-public-ip-on-launch
Expect: no output. New instances in sn-web-A will now get a public IPv4 (once routing is added in L4).
Check usable IP count
aws ec2 describe-subnets --subnet-ids $WEBA \
  --query "Subnets[0].{Cidr:CidrBlock,Free:AvailableIpAddressCount}"
Expect: a /20 shows ~4091 free (4096 − 5 reserved).
Fix / Key infoAuto-assign public IP makes a subnet "public-capable", but it isn't truly public until it also has a route to an internet gateway (Lesson 4).
Clean upKeep these for Lesson 4. To remove now: aws ec2 delete-subnet --subnet-id $WEBA (repeat for each).

Routing, Internet Gateway & Bastion

Make a subnet public and reach the internet.

Quick Understanding

ConceptMeaningExam relevance
Route tableRules deciding where a subnet's traffic goes.One RT per subnet.
Internet Gateway (IGW)VPC's door to the internet; does NAT for public IPv4.One per VPC, HA, regional.
Public subnetAuto-assign IP + route 0.0.0.0/0 → IGW.Definition trap.
Bastion / jump boxPublic EC2 used to reach private instances.Know it's legacy.

Key Terms

TermSimple meaning
RouteOne rule: "to reach X, send via Y".
Local routeBuilt-in route for the VPC CIDR; can't be removed.
AssociationLinking a route table to a subnet.
0.0.0.0/0"Everywhere else" — the default route to the internet.
Bastion hostA gateway server you log into first, then hop inward.

Explanation

graph TD
  EC2[Instance in web subnet] --> RT[Route Table]
  RT -->|10.16.0.0/16 = local| VPC[Inside VPC]
  RT -->|0.0.0.0/0| IGW[Internet Gateway]
  IGW --> NET[Internet]
  IGW -. maps public IP to private IP .-> EC2
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class IGW k;
  • Each subnet uses one route table; routes pick the most specific match (longest prefix).
  • The local route (the VPC CIDR) is always there and can't be removed.
  • IGW is one-per-VPC, highly available, regional. It performs static NAT: maps an instance's public IPv4 ↔ its private IP.
  • The instance OS only ever sees its private IP — the IGW does the public translation.
  • Public subnet = auto-assign public IP and a default route to the IGW.
  • Bastion host: a public jump box to reach private instances. It's legacy — prefer SSM Session Manager. [added for exam]

Demo

Demo · CLI v2Situation: make sn-web-A public and launch a bastion. Check first: $VPC and $WEBA from L2/L3 set. Note: launches a t3.micro EC2 (free tier) — clean up after.
Create + attach an internet gateway
IGW=$(aws ec2 create-internet-gateway \
  --tag-specifications 'ResourceType=internet-gateway,Tags=[{Key=Name,Value=a4l-igw}]' \
  --query InternetGateway.InternetGatewayId --output text)
aws ec2 attach-internet-gateway --internet-gateway-id $IGW --vpc-id $VPC
echo $IGW
Expect: an IGW ID; attach gives no output.
Create a public route table + default route to IGW
RT=$(aws ec2 create-route-table --vpc-id $VPC \
  --tag-specifications 'ResourceType=route-table,Tags=[{Key=Name,Value=a4l-rt-public}]' \
  --query RouteTable.RouteTableId --output text)
aws ec2 create-route --route-table-id $RT \
  --destination-cidr-block 0.0.0.0/0 --gateway-id $IGW
aws ec2 associate-route-table --route-table-id $RT --subnet-id $WEBA
Expect: create-route returns "Return": true; association returns an ID. sn-web-A is now public.
Security group allowing SSH
SG=$(aws ec2 create-security-group --group-name a4l-bastion-sg \
  --description "bastion ssh" --vpc-id $VPC --query GroupId --output text)
aws ec2 authorize-security-group-ingress --group-id $SG \
  --protocol tcp --port 22 --cidr 0.0.0.0/0   # demo only - tighten to your IP!
Expect: an SG ID; ingress rule added. (In real life allow only your own IP, not 0.0.0.0/0.)
Launch a bastion (free-tier) instance
AMI=$(aws ssm get-parameters \
  --names /aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64 \
  --query "Parameters[0].Value" --output text)
aws ec2 create-key-pair --key-name a4l-key \
  --query KeyMaterial --output text > a4l-key.pem && chmod 400 a4l-key.pem
INST=$(aws ec2 run-instances --image-id $AMI --instance-type t3.micro \
  --key-name a4l-key --subnet-id $WEBA --security-group-ids $SG \
  --query "Instances[0].InstanceId" --output text)
echo $INST
Expect: an instance ID. Verify the free-tier type for your region (t2.micro vs t3.micro).
Get its public IP and connect
aws ec2 describe-instances --instance-ids $INST \
  --query "Reservations[0].Instances[0].PublicIpAddress" --output text
# ssh -i a4l-key.pem ec2-user@THAT_PUBLIC_IP
Expect: a public IPv4; SSH connects → you have a working public VPC + bastion.
Fix / Key infoThe instance never "sees" its public IP — the IGW maps it. SSH-open-to-the-world bastions are a classic bad practice; the exam wants you to recognise this and prefer SSM.
Clean up
aws ec2 terminate-instances --instance-ids $INST
aws ec2 wait instance-terminated --instance-ids $INST
aws ec2 delete-key-pair --key-name a4l-key ; rm -f a4l-key.pem
aws ec2 delete-security-group --group-id $SG
aws ec2 disassociate-route-table --association-id $(aws ec2 describe-route-tables \
  --route-table-ids $RT --query "RouteTables[0].Associations[0].RouteTableAssociationId" --output text)
aws ec2 delete-route-table --route-table-id $RT
aws ec2 detach-internet-gateway --internet-gateway-id $IGW --vpc-id $VPC
aws ec2 delete-internet-gateway --internet-gateway-id $IGW
Expect: instance terminated, then all networking removed. Keep $VPC/subnets if continuing to L8 (else delete subnets + VPC too).

Stateful vs Stateless Firewalls

Why return traffic is the whole story.

Quick Understanding

ConceptMeaningExam relevance
StatefulRemembers a connection; return traffic is auto-allowed.= Security Groups.
StatelessEach direction judged alone; must allow responses too.= NACLs.
Ephemeral portsRandom high ports used for replies (1024–65535).NACL outbound rules.

Key Terms

TermSimple meaning
Connection stateKnowing a reply belongs to a request you already allowed.
Ephemeral portA temporary high-numbered port a client uses to receive a reply.
Inbound / outboundTraffic coming in / going out.

Explanation

graph TD
  REQ[Request out on port 443] --> RESP[Reply comes back on a random high port]
  RESP --> SF{Firewall type?}
  SF -->|Stateful SG| AUTO[Reply auto-allowed - remembers the request]
  SF -->|Stateless NACL| MANUAL[Must add a rule for the ephemeral reply port]
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class AUTO,MANUAL k;
  • Stateful firewalls track connections — allow a request and the reply is automatic.
  • Stateless firewalls judge each packet alone — you must also allow the return traffic.
  • Replies usually come back on ephemeral ports (1024–65535), so stateless rules must allow that range.
  • In AWS: Security Groups = stateful, NACLs = stateless.

Network Access Control Lists (NACLs)

Stateless firewall at the subnet edge.

Quick Understanding

ConceptMeaningExam relevance
NACLOptional stateless firewall around one or more subnets.Subnet-level control.
Numbered rulesEvaluated low → high; first match wins.Order matters.
Can DENYNACLs can explicitly block (SGs can't).Block bad IPs.
StatelessNeed inbound + outbound (ephemeral) rules.Return traffic trap.

Key Terms

TermSimple meaning
Rule numberThe order position; lower numbers are checked first.
Explicit denyA rule that blocks specific traffic.
Default NACLComes allowing all traffic; a custom NACL denies all until you add rules.

Explanation

graph TD
  PKT[Packet enters subnet] --> N1[Rule 100?]
  N1 -->|match| ACT[Allow or Deny - stop]
  N1 -->|no match| N2[Rule 200?]
  N2 -->|no match| STAR[Rule * = DENY all]
  classDef w fill:#3a151c,stroke:#fb7185,color:#e7edf3;
  class STAR w;
  • NACLs sit at the subnet boundary and are stateless (need inbound + outbound rules).
  • Rules are numbered; lowest number that matches wins, then it stops.
  • NACLs can explicitly deny — useful to block a specific IP/range.
  • A subnet has one NACL; a NACL can cover many subnets. They can't reference SGs or AWS resources — only IP ranges/ports.

Demo

Demo · CLI v2Situation: create a NACL, allow web + ephemeral, attach to sn-web-A. Check first: $VPC and $WEBA set.
Create the NACL
NACL=$(aws ec2 create-network-acl --vpc-id $VPC \
  --tag-specifications 'ResourceType=network-acl,Tags=[{Key=Name,Value=a4l-web-nacl}]' \
  --query NetworkAcl.NetworkAclId --output text)
echo $NACL
Expect: a NACL ID. A new custom NACL denies everything until rules are added.
Allow inbound HTTPS + outbound ephemeral (stateless!)
aws ec2 create-network-acl-entry --network-acl-id $NACL --rule-number 100 \
  --protocol tcp --port-range From=443,To=443 --cidr-block 0.0.0.0/0 \
  --rule-action allow --ingress
aws ec2 create-network-acl-entry --network-acl-id $NACL --rule-number 100 \
  --protocol tcp --port-range From=1024,To=65535 --cidr-block 0.0.0.0/0 \
  --rule-action allow --egress
Expect: no output. Inbound 443 allowed; outbound ephemeral ports allowed so replies can leave.
Attach the NACL to the web subnet
ASSOC=$(aws ec2 describe-network-acls \
  --filters Name=association.subnet-id,Values=$WEBA \
  --query "NetworkAcls[0].Associations[?SubnetId=='$WEBA'].NetworkAclAssociationId" --output text)
aws ec2 replace-network-acl-association --association-id $ASSOC --network-acl-id $NACL
Expect: a new association ID — the subnet now uses your NACL.
Fix / Key infoBecause NACLs are stateless, the outbound ephemeral rule is essential — without it, allowed inbound requests get no reply.
Clean upRe-attach the default NACL, then delete yours:
DEF=$(aws ec2 describe-network-acls --filters Name=vpc-id,Values=$VPC \
  Name=default,Values=true --query "NetworkAcls[0].NetworkAclId" --output text)
NEWASSOC=$(aws ec2 describe-network-acls --network-acl-ids $NACL \
  --query "NetworkAcls[0].Associations[?SubnetId=='$WEBA'].NetworkAclAssociationId" --output text)
aws ec2 replace-network-acl-association --association-id $NEWASSOC --network-acl-id $DEF
aws ec2 delete-network-acl --network-acl-id $NACL
Expect: subnet back on the default NACL; custom NACL deleted.

Exam Pointers

Exam Tip
  • NACL = stateless, subnet level, numbered rules (low wins).
  • Only NACLs can explicitly DENY (e.g. block one IP).
  • NACLs work on IP/port only — they can't reference SGs or AWS resources.
  • Need both inbound and outbound (ephemeral) rules.

Yes — unlike Security Groups.

Stateful firewall attached to resources.

Quick Understanding

ConceptMeaningExam relevance
Security GroupStateful firewall on an ENI (a resource), not a subnet.Resource-level control.
Allow-onlySGs can only allow; there's an implicit deny.Can't block one IP.

Key Terms

TermSimple meaning
ENIElastic Network Interface — a virtual network card on a resource.

Explanation

graph TD
  WEBSG[web-SG: allow 443 from anywhere] --> WEB[Web instances]
  APPSG[app-SG: allow 8080 FROM web-SG] --> APP[App instances]
  WEB -->|traffic on 8080| APP
  NOTE[SG reference auto-follows instances
even when their IPs change]
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class WEBSG,APPSG k;
  • SGs attach to resources (ENIs), are stateful (replies automatic), and are allow-only.
  • Because there's an implicit deny and no explicit deny, SGs can't block a single IP — use a NACL for that.
  • Rules can reference another SG (or itself), so they keep working as instances scale and IPs change.
  • Often paired with NACLs: SGs for normal allow-rules, NACLs for explicit blocks.

Demo

Demo · CLI v2Situation: make a web SG and an app SG, where app only accepts traffic FROM the web SG. Check first: $VPC set.
Create both security groups
WEBSG=$(aws ec2 create-security-group --group-name a4l-web-sg \
  --description "web" --vpc-id $VPC --query GroupId --output text)
APPSG=$(aws ec2 create-security-group --group-name a4l-app-sg \
  --description "app" --vpc-id $VPC --query GroupId --output text)
echo "web=$WEBSG app=$APPSG"
Expect: two SG IDs.
Allow HTTPS to web from anywhere
aws ec2 authorize-security-group-ingress --group-id $WEBSG \
  --protocol tcp --port 443 --cidr 0.0.0.0/0
Expect: rule added (JSON shows the rule).
Allow app port 8080 ONLY from the web SG (reference)
aws ec2 authorize-security-group-ingress --group-id $APPSG \
  --protocol tcp --port 8080 --source-group $WEBSG
Expect: rule added. App accepts 8080 only from instances that have the web SG — no IPs to maintain.
Fix / Key infoSG references are the clean way to wire tiers together. They keep working as Auto Scaling adds/removes instances.
Clean up
# delete the referencing SG first, then the referenced one
aws ec2 delete-security-group --group-id $APPSG
aws ec2 delete-security-group --group-id $WEBSG
Expect: both removed (order matters because of the reference).

NAT & NAT Gateway

Let private instances reach out, without being reachable.

Quick Understanding

ConceptMeaningExam relevance
NATMany private IPs share one public IP for outbound traffic.Outbound only.
NAT GatewayAWS-managed NAT in a public subnet; needs an Elastic IP.HA per AZ.
Per-AZ HAOne NAT GW per AZ for resilience.Design trap.
Costs moneyHourly + data charges (not free tier).Delete after demo!

Key Terms

TermSimple meaning
NATNetwork Address Translation — swaps private IPs for a shared public IP.
Elastic IP (EIP)A fixed public IPv4 you own.
Egress-only IGWThe IPv6 equivalent of NAT (outbound-only).
NAT instanceAn older DIY NAT on an EC2 instance.

Explanation

graph TD
  PRIV[Private instance 10.16.x] --> RT[Private route table]
  RT -->|0.0.0.0/0| NATGW[NAT Gateway in PUBLIC subnet + EIP]
  NATGW --> IGW[Internet Gateway]
  IGW --> NET[Internet]
  NET -. cannot start a connection inward .-x PRIV
  classDef k fill:#0e2647,stroke:#60a5fa,color:#e7edf3;
  class NATGW k;
  • NAT lets private instances make outbound connections (updates, APIs) but blocks inbound from the internet.
  • NAT Gateway sits in a public subnet, needs an Elastic IP, and private subnets route 0.0.0.0/0 to it.
  • A NAT GW is resilient only within its AZ → put one per AZ for high availability.
  • It's managed and scales, but costs money (hourly + data) — not free tier.
  • A NAT GW can't have a security group (it uses the subnet's NACL). For IPv6, use an egress-only IGW instead. [added for exam]

Demo

Cost warningA NAT Gateway is NOT free (~hourly charge + data). Do this quickly and run the cleanup straight after.
Demo · CLI v2Situation: give a private subnet outbound internet via a NAT GW. Check first: $VPC, a public subnet ($WEBA with IGW route from L4), and a private subnet ($APPA from L3).
Allocate an Elastic IP
EIP=$(aws ec2 allocate-address --domain vpc --query AllocationId --output text)
echo $EIP
Expect: an allocation ID like eipalloc-....
Create the NAT Gateway in the PUBLIC subnet
NATGW=$(aws ec2 create-nat-gateway --subnet-id $WEBA \
  --allocation-id $EIP --query NatGateway.NatGatewayId --output text)
aws ec2 wait nat-gateway-available --nat-gateway-ids $NATGW
echo $NATGW
Expect: a NAT GW ID; the wait finishes when it's available (a minute or two).
Private route table → NAT GW, attach to private subnet
PRT=$(aws ec2 create-route-table --vpc-id $VPC \
  --tag-specifications 'ResourceType=route-table,Tags=[{Key=Name,Value=a4l-rt-private}]' \
  --query RouteTable.RouteTableId --output text)
aws ec2 create-route --route-table-id $PRT \
  --destination-cidr-block 0.0.0.0/0 --nat-gateway-id $NATGW
aws ec2 associate-route-table --route-table-id $PRT --subnet-id $APPA
Expect: route created ("Return": true) and associated. Instances in sn-app-A can now reach the internet outbound (e.g. yum update) but stay unreachable inbound.
Fix / Key infoFor real HA you'd create a NAT GW in each AZ and point each AZ's private route table at its local NAT GW.
Clean up (do this now — NAT GW bills hourly)
aws ec2 delete-nat-gateway --nat-gateway-id $NATGW
aws ec2 wait nat-gateway-deleted --nat-gateway-ids $NATGW
aws ec2 release-address --allocation-id $EIP
ASSOC=$(aws ec2 describe-route-tables --route-table-ids $PRT \
  --query "RouteTables[0].Associations[0].RouteTableAssociationId" --output text)
aws ec2 disassociate-route-table --association-id $ASSOC
aws ec2 delete-route-table --route-table-id $PRT
Expect: NAT GW deleted, EIP released (stops charges), route table removed. Then tear down any remaining subnets/VPC if you're finished.
